Prevejas tendências futuras usando dados históricos. Encaminhe mensagens, detecte spam e recomende produtos para os usuários, usando a Google Prediction API. A poderosa Infraestrutura do Google analisa os dados para você.

Rápido e confiável

Seus dados são replicados em vários centros de dados usando o Google Cloud Storage. A maioria das consultas de previsão leva menos de 200 ms. Maior desempenho está disponível. Consulte a documentação para obter dicas de desempenho.

Integração na nuvem

Os serviços do Google foram desenvolvidos para trabalharem em conjunto. Use os resultados do Google BigQuery ou puxe conjuntos de dados de até 2,5 GB do Google Clould Storage para criar um modelo previsivo. Use a Google Prediction API diretamente do Google App Engine.

Poderosas ferramentas de desenvolvimento

A Google Prediction API pode se integrar com o Google App Engine, e a API RESTful está disponível por meio de bibliotecas para muitas linguagens populares, como Python, JavaScript e. NET. Consulte a documentação para obter a lista completa.

Preços

Cota gratuita

O uso é gratuito durante os primeiros seis meses, até os seguintes limites por projeto do Google Cloud Console. Essa cota gratuita aplica-se mesmo quando o faturamento está ativado, até que o tempo de validade de seis meses.

  • Limites de uso:
    • Previsões: 100 predições/dia
    • Previsões de modelos hospedados: modelos hospedados têm um limite de uso de 100 previsões/dia/usuário em todos os modelos.
    • Treinamento : 5 MB treinados/dia
    • Atualizações em fluxo contínuo:100 atualizações em fluxo contínuo/dia
  • Limite total: 20.000 previsões
  • Expiração: a cota gratuita expira seis meses após a ativação do Google Prediction para seu projeto no Google Cloud Console.

Alguns cenários de uso da Google Prediction API dependem do Google Cloud Storage, que tem diferentes condições de preços da Google Prediction API.

Uso pago

O uso pago vem com um contrato de nível de serviço de disponibilidade de 99,9%. Leia mais sobre o SLA aqui.

Monitore uso do Google Cloud Console. Qualquer uso além da cota gratuita de uso falha se o faturamento não está ativado para o projeto. Quando a cota gratuita é ultrapassada, os seguintes preços e limites se aplicam:

  • Taxa base: taxa mensal de US$ 10 por projeto do Google Cloud Console.
  • Tarifas de uso:
    • Previsão:
      • 10.000 previsões/mês: US$ 0,00
      • Mais de 10.001 previsões/mês: US$ 0,50/1000 previsões além das 10.000 iniciais
    • Treinamento:
      • US$ 0,002/MB de treinamento em massa (tamanho máximo de cada conjunto de dados: 2,5 GB)
      • 0-10.000 atualizações de fluxo contínuo: US$ 0,00
      • Mais de 10.001 atualizações de fluxo contínuo: US$ 0,05/1.000 previsões além das 10.000 iniciais
  • Limites de uso:
    • Se você pretende fazer mais de 40.000 previsões/dia, fale conosco.
    • A Google Prediction API tem um limite padrão de 2.000.000 previsões/dia por projeto para seus próprios modelos.
    • Modelos hospedados têm um limite de uso de 100 previsões/dia/usuário em todos os modelos de demonstração. Os desenvolvedores podem definir limites de uso mais altos pagos para os modelos que eles criam. Consulte a documentação de cada modelo para obter mais detalhes.

Importante: Esses encargos não incluem quaiquer taxas do Google Cloud Storage que possam ser necessárias para manter os dados de treinamento. Consulte a lista de preços do Google Cloud Storage para identificar quaisquer custos adicionais que possam incorrer.

  • O que é a API do Google Prediction?

    Ferramentas de aprendizado de máquina do Google baseadas em nuvem podem ajudar a analisar dados para adicionar os seguintes recursos a seus aplicativos: análise de opiniões do cliente, detecção de spam, decisões de roteamento de mensagens, análise de oportunidade Upsell, classificação de documentos e e-mail e muito mais.

  • Como prever compras

    Imagine um site que vende cerveja, vinho e queijo, e você quer prever se um visitante estará interessado em vinho, considerando o histórico de compras dele. Nessa situação, podemos criar dados de treinamento com três elementos: o número de vezes que o cliente comprou sobremesa de queijo. Um valor de 1 se o cliente comprou vinho alguma vez e de 0, caso não tenha comprado. O número de vezes que o cliente comprou cerveja.

  • Como criar um modelo de análise de opiniões

    Este documento explica como criar um modelo básico de análise de opiniões, usando o API do Google Prediction. Um modelo de análise de opiniões é usado para analisar uma string de texto e classificá-la com um dos marcadores que você fornece. Por exemplo, você pode analisar um tweet para determinar se ele é positivo ou negativo ou analisar um e-mail para identificar se ele é feliz, frustrado ou triste.
