Cloud Endpoints

Créez des services RESTful accessibles depuis les clients iOS, Android et JavaScript. Facilitez la connexion frontend en générant automatiquement des bibliothèques clientes. Bénéficiez de fonctionnalités intégrées comme la protection contre les attaques par déni de service, la compatibilité avec OAuth 2.0 et la gestion des clés des clients.

Fonctionnalités

Un seul outil pour plusieurs clients

Générez des bibliothèques clientes pour Android, iOS et le Web, le tout à partir d'une seule source. Avec Cloud Endpoints, votre code est mis en forme de façon à créer un serveur d'API en quelques étapes seulement. Des bibliothèques d'API sont disponibles pour les langages Java, Python, Go et PHP. Pour en savoir plus sur Cloud Endpoints, consultez la documentation, ou lancez-vous en essayant notre exemple d'application Web, Tic Tac Toe.

Extension de l'infrastructure d'App Engine

Tous les outils et les bibliothèques d'App Engine sont désormais accessibles depuis les appareils mobiles. Accédez à Datastore, à Cloud Storage et aux files d'attente de tâches à l'aide de votre backend App Engine, sans aucun travail supplémentaire de votre part. Intégrez rapidement l'authentification OAuth 2.0 en vous référant à nos exemples.

Architecture client-serveur à maintenance réduite

Cloud Endpoints repose sur App Engine. Par conséquent, il offre la même évolutivité simple et rapide, et vous n'avez pas besoin de vous soucier de la maintenance des serveurs ni de l'équilibrage des charges. Comme pour App Engine et nos autres services de cloud, vous ne payez que ce que vous consommez.

Intégration flexible côté client

Annotez votre API côté serveur, puis générez automatiquement des bibliothèques clientes destinées à Android et à iOS. Proposez des clients Web standards à l'aide d'une bibliothèque cliente JavaScript minimale. Tous vos clients utilisent des API similaires et reposent sur le même backend, ce qui réduit le temps de développement.

Tarifs

Cloud Endpoints est disponible gratuitement pour les applications que vous exécutez sur App Engine et Compute Engine.

Sources d'information

  • Créer une API Endpoints

    Une API Endpoints est un service d'appel de procédure à distance (RPC, Remote Procedure Call) qui fournit des méthodes distantes accessibles depuis les clients externes. Chaque API Endpoints se compose d'une classe de service RPC qui correspond à une sous-classe de la classe ProtoRPC remote.Service, ainsi que d'une ou plusieurs méthodes.

  • Générer des bibliothèques clientes

    Une fois que vous avez créé des API Endpoints, vous pouvez générer des bibliothèques clientes pour Android et iOS.

  • Utiliser Endpoints depuis un client iOS

    Pour utiliser Endpoints depuis une application iOS, vous devez suivre plusieurs étapes. D'abord, compilez le générateur de bibliothèques clientes et générez la bibliothèque. Ensuite, ajoutez les fichiers requis à votre projet iOS. Créez l'objet Service devant permettre d'envoyer des requêtes. Enfin, appelez l'API Endpoints. Une étape facultative permet d'assurer l'authentification des appels.
