Cloud Datastore

Cloud Datastore est une base de données gérée NoSQL sans schéma permettant de stocker les données non relationnelles. Elle évolue automatiquement en fonction de vos besoins et gère aussi bien les transactions que les requêtes robustes de type SQL.

Essayer maintenant

Fonctionnalités

Accès sans schéma avec requêtes de type SQL

Ne vous souciez pas des migrations ni des modèles de données. Cloud Datastore est un service de stockage sans schéma. Il élimine les contraintes liées à la structure sous-jacente des données, ce qui vous dote d'une plus grande agilité. Un moteur de requête robuste vous permet de rechercher des données parmi plusieurs propriétés et de les trier en fonction de vos besoins.

Base de données gérée

Cloud Datastore est une base de données entièrement gérée. La segmentation et la réplication des données sont automatisées. Vous disposez ainsi d'une base de données cohérente et haute disponibilité.

Évolution automatique en fonction des utilisateurs

Cloud Datastore s'adapte automatiquement à vos besoins. Vous pouvez vous concentrer sur le développement de votre application sans vous préoccuper de l'allocation des ressources ni de l'anticipation des charges de travail.

Transactions ACID

Cloud Datastore permet les transactions ACID dans le cadre d'un contrôle de simultanéité optimiste. Dans une transaction unique, votre application peut exécuter plusieurs opérations de magasin de données, qui réussissent ou échouent toutes, garantissant ainsi l'intégrité des données.

Redondance intégrée

Un seul appel d'API déclenche la réplication automatique des données dans plusieurs centres de données. La haute disponibilité et la durabilité sont intégrées au cœur même du service.

Outils de développement en local

Grâce au kit de développement de Cloud Datastore, vous pouvez développer des applications, les tester et créer des itérations en local, sans qu'un déploiement complet soit nécessaire.

Données accessibles en tout lieu

Développez des solutions qui utilisent à la fois App Engine et Compute Engine en vous appuyant sur Cloud Datastore comme point d'intégration. L'interface RESTful disponible dans Cloud Datastore permet d'accéder facilement aux données depuis n'importe quelle cible de déploiement.

Tarifs

Les frais et le seuil de gratuité applicables à Google Cloud Datastore sont les mêmes que les frais de magasin de données d'App Engine, qui sont détaillés ci-dessous. Sachez que Cloud Datastore comprend un composant serveur qui s'exécute sur Google App Engine. Des frais horaires d'utilisation de l'instance sont applicables en conséquence.

Sources d'information

  • Présentation de Google Cloud Datastore

    Grâce à un ensemble d'outils et de bibliothèques clientes, il est possible d'accéder à Google Cloud Datastore et de le gérer depuis des clients s'exécutant sur des instances Google Cloud Compute Engine ou autres. L'API basée sur le protocole RPC est hébergée au sein du même service Google que les API Google Maps, YouTube et Google Apps. Elle est compatible avec la plupart des applications et des langages. Par exemple, vous pouvez l'utiliser avec une application Web Node.js qui s'exécute sur une machine virtuelle ou un serveur d'applications Java sur site.

  • Activer Google Cloud Datastore

    Cet article explique comment activer l'API Google Cloud Datastore. Il décrit les quotas qui lui sont associés et indique comment activer la facturation. La procédure d'activation de l'API dépend de la configuration et de l'environnement en place.

  • Concepts clés de Datastore

    Datastore est un magasin de données sans schéma basé sur des objets. Vous bénéficiez d'un stockage robuste et évolutif pour vos applications grâce aux fonctionnalités suivantes : aucune interruption planifiée, transactions atomiques, haute disponibilité des lectures et des écritures, cohérence forte pour les lectures et pour les requêtes ascendantes, et cohérence éventuelle pour toutes les autres requêtes.
