Fonctionnalités

Exploitation des données

Prévoyez les tendances futures en fonction des données historiques. Routez les messages, détectez le spam et recommandez des produits aux utilisateurs à l'aide de Prediction API. Les données sont analysées pour vous au sein de la puissante infrastructure de Google.

Rapidité et fiabilité

Vos données sont répliquées dans plusieurs centres de données à l'aide de Cloud Storage. La plupart des requêtes de prédiction sont traitées en moins de 200 millisecondes, et il est possible d'atteindre des performances encore supérieures. Pour obtenir des conseils à ce sujet, consultez la documentation.

Intégration dans le cloud

Les services Google sont conçus pour fonctionner ensemble. Créez un modèle prédictif à partir des résultats de Google BigQuery ou d'ensembles de données extraits de Cloud Storage pouvant atteindre 2,5 Go. Vous pouvez utiliser Prediction API directement dans Google App Engine.

Outils de développement puissants

Prediction API s'intègre avec App Engine. L'API RESTful est disponible dans les bibliothèques des langages les plus courants, tels que Python, JavaScript et .NET. Consultez la liste complète dans la documentation.

Exemples et assistance

Pour en savoir plus sur Prediction API, consultez nos exemples d'applications. Inspirez-vous de nos démos et de notre sélection d'applications.

Vous bénéficiez d'une utilisation limitée gratuite pendant les six premiers mois. Au-delà, la tarification claire de Prediction API fait l'objet d'un contrat de niveau de service qui garantit une disponibilité à 99,9 %. Comme pour les autres produits Cloud, vous ne payez que ce que vous consommez.

Tarifs

Quota gratuit

L'utilisation est gratuite pendant les six premiers mois, dans les limites suivantes, qui s'entendent par projet Google Cloud Console. Ce quota gratuit s'applique même si la facturation est activée, jusqu'à l'expiration du délai de six mois.

  • Limites d'utilisation :
    • Prédictions : 100 prédictions/jour
    • Prédictions à partir des modèles hébergés : limite de 100 prédictions par jour et par utilisateur sur l'ensemble des modèles hébergés
    • Apprentissage : 5 Mo de données d'apprentissage par jour
    • Mises à jour en streaming : 100 mises à jour en streaming/jour
  • Limite sur la durée totale : 20 000 prédictions
  • Expiration : expiration du quota gratuit six mois après l'activation de Google Prediction pour votre projet dans Google Cloud Console

Remarque : Dans certains cas, le service Google Cloud Storage est nécessaire pour utiliser Prediction API. Ce service fait l'objet de conditions tarifaires distinctes.

Surveillez votre consommation dans Google Cloud Console. Si vous n'avez pas activé la facturation pour votre projet, toute utilisation au-delà du quota gratuit échoue. Les limites et les prix suivants s'appliquent au-delà du quota gratuit :

  • Frais de base : 10 USD par mois et par projet Google Cloud Console

Important : Ces tarifs n'incluent pas les frais du service Google Cloud Storage éventuellement requis pour le stockage des données d'apprentissage. Pour déterminer les autres frais applicables, consultez la grille tarifaire de Google Cloud Storage.

Sources d'information

  • Qu'est-ce que Google Prediction API ?

    Basés sur le cloud, les outils d'apprentissage automatique de Google vous permettent d'analyser vos données dans le but d'ajouter les fonctionnalités suivantes à vos applications : analyse de l'avis des clients, détection du spam, choix du routage des messages, analyse des opportunités de vente de produits de gamme supérieure, classification des documents et des e-mails, etc.

  • Prédire les achats

    Imaginez un site de vente de bières, de vins et de fromages. Vous souhaitez savoir si un visiteur est susceptible d'acheter du vin d'après l'historique de ses achats. Dans une telle situation, vous pouvez créer des données d'apprentissage qui répertorient trois caractéristiques : le nombre de fois où le client a acheté des fromages fins, une valeur indiquant si le client a déjà acheté du vin (1 pour oui, 0 pour non) et le nombre de fois où le client a acheté de la bière.

  • Créer un modèle d'analyse des sentiments

    Ce document explique comment créer un modèle de base d'analyse des sentiments des utilisateurs à l'aide de Google Prediction API. Un tel modèle permet d'analyser une chaîne de texte et de la classer selon les libellés que vous fournissez. Par exemple, vous pouvez analyser un tweet et déterminer s'il est positif ou négatif, ou analyser un e-mail et déterminer s'il exprime de la joie, de la frustration ou de la tristesse.
