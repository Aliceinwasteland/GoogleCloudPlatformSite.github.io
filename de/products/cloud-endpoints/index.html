Cloud Endpoints

Erstellen Sie RESTful-Dienste und stellen Sie sie für Android-, iOS- und JavaScript-Clients bereit. Erzeugen Sie automatisch Client-Bibliotheken, um das Anschließen des Front-End zu vereinfachen. Die integrierten Funktionen bieten DoS-Schutz (Denial-of-Service), Unterstützung für OAuth 2.0 und Verwaltung von Client-Schlüsseln.

Funktionen

Ein Tool, mehrere Clients

Erstellen Sie Client-Bibliotheken für Android, iOS und webbasierte Clients aus einer einzigen Quelle. Cloud Endpoints umschließt Ihren Code, sodass Sie in nur wenigen Schritten einen API-Server erstellen können. Cloud Endpoint API-Bibliotheken sind in Java, Python, PHP und Go verfügbar. Weitere Informationen zu Cloud Endpoints finden Sie in unserer Dokumentation. Sie können auch direkt einsteigen und unsere Beispiel-Webanwendung Tic Tac Toe ausprobieren.

App Engine-Infrastruktur erweitern

Alle in App Engine verfügbaren Tools und Bibliotheken stehen jetzt auch für Ihre Mobilgeräte zur Verfügung. Greifen Sie ohne zusätzlichen Aufwand über Ihr App Engine-Back-End auf Datastore, Cloud Storage und Aufgabenwarteschlangen zu. Anhand unserer Beispiele können Sie schnell eine OAuth 2.0-Authentifizierung integrieren.

Client-Server mit geringem Wartungsaufwand

Da Cloud Endpoints von App Engine unterstützt wird, brauchen Sie keine Server zu unterhalten und keinen Lastenausgleich durchzuführen und profitieren von derselben schnellen und einfachen Skalierung. Wie bei App Engine und unseren anderen Clouddiensten zahlen Sie nur für die tatsächliche Nutzung.

Flexible clientseitige Integration

Kommentieren Sie Ihre serverseitige API und erstellen Sie dann automatisch Ihre Client-Bibliotheken. Client-Bibliotheken werden für Android und iOS erstellt. Bringen Sie Standard-Web-Clients mit einer minimalen JavaScript-Client-Bibliothek zum Laufen. Da alle Ihre Kunden ähnliche APIs und dasselbe Back-End verwenden, bleibt die Entwicklungszeit gering.

Preise

Cloud Endpoints ist für Anwendungen, die auf App Engine und Compute Engine ausgeführt werden, kostenlos verfügbar.

Ressourcen

  • Endpoints API erstellen

    Eine Endpoints API ist ein RPC-Dienst (remote procedure call – Remoteprozeduraufruf), der für externe Clients zugängliche Remote-Methoden bietet. Jede Endpoints API besteht aus einer RPC-Dienstklasse, die eine Unterklasse der ProtoRPC-remote.Service-Klasse erstellt, sowie aus einer oder mehreren Methoden.

  • Client-Bibliotheken erzeugen

    Nach dem Erstellen Ihrer Endpunkte können Sie die Client-Bibliotheken für Android und iOS generieren.

  • Endpunkte in einem iOS-Client verwenden

    Gehen Sie zur Verwendung von Endpunkten in einer iOS-App wie folgt vor: Kompilieren Sie den Generator für Client-Bibliotheken und erzeugen Sie Ihre Bibliothek. Fügen Sie zu Ihrem iOS-Projekt die erforderlichen Dateien hinzu. Erstellen Sie das Dienstobjekt für die Durchführung von Anfragen. Rufen Sie die vom Endpunkt bereitgestellte API auf. Fügen Sie bei Bedarf Unterstützung für authentifizierte Aufrufe hinzu.
