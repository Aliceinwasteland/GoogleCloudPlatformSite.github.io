Cloud Datastore — Google Cloud Platform

Cloud Datastore

Verwenden Sie eine verwaltete schemalose NoSQL-Datenbank zum Speichern nicht relationaler Daten. Cloud Datastore sorgt für eine automatische Skalierung entsprechend Ihrem Bedarf und unterstützt Transaktionen und leistungsfähige, SQL-ähnliche Abfragen.

Jetzt testen

Funktionen

Schemaloser Zugriff mit SQL-ähnlichen Abfragen

Sie brauchen sich nicht um Datenmodelle und die Migration zu kümmern. Cloud Datastore ist ein schemaloser Speicherdienst, der Sie noch produktiver macht, weil Sie nicht mehr über die zugrunde liegende Struktur der Daten nachdenken müssen. Cloud Datastore bietet ein stabiles Abfragemodul, mit dem Sie in mehreren Eigenschaften nach Daten suchen und bei Bedarf eine Sortierung durchführen können.

Verwaltete Datenbank

Cloud Datastore wird vollständig verwaltet. Google führt automatisch Fragmentierung und Replikation durch, um Ihnen eine konsistente Datenbank mit hoher Verfügbarkeit zu bieten.

Automatische Skalierung

Cloud Datastore führt automatisch eine Skalierung entsprechend Ihren Anforderungen durch. So können Sie sich auf das Erstellen der App konzentrieren und brauchen sich nicht um Bereitstellung und Lastvorhersage zu kümmern.

ACID-Transaktionen

Cloud Datastore bietet ACID-Transaktionen mithilfe einer optimistischen Gleichzeitigkeitserkennung. Ihre App kann mehrere Datenspeicheroperationen in einer einzigen Transaktion ausführen. Zur Wahrung der Integrität Ihrer Daten werden dabei entweder alle abgeschlossen oder es schlagen alle fehl.

Integrierte Redundanz

Mit einem einzigen API-Aufruf werden Daten automatisch in mehreren Rechenzentren repliziert. Hohe Verfügbarkeit und Langlebigkeit sind integriert.

Lokale Entwicklungstools

Mit dem Cloud Datastore Development Kit können Sie Ihre Apps ohne vollständige Bereitstellung lokal entwickeln, testen und optimieren.

Datenzugriff von jedem Standort aus

Erstellen Sie Lösungen, die App Engine und Compute Engine nutzen und Cloud Datastore als Integrationspunkt verwenden. Dank der von Cloud Datastore bereitgestellten RESTful-Benutzeroberfläche kann jedes Bereitstellungsziel ganz einfach auf die Daten zugreifen.

Preise

Die Gebühren und kostenlosen Kontingente für Google Cloud Datastore sind dieselben wie die unten aufgeführten Datenspeichergebühren von App Engine. Beachten Sie, dass Cloud Datastore eine auf Google App Engine basierende Bereitstellungskomponente umfasst. Deshalb fallen Kosten für Instanzstunden an.

Ressourcen

  • Übersicht über Google Cloud Datastore

    Der Google Cloud Datastore bietet eine Reihe von Tools und Client-Bibliotheken, mit denen Google Cloud Datastore über Clients genutzt und verwaltet werden kann, die auf Google Cloud Compute Engine-Instanzen oder anderswo ausgeführt werden. Er bietet eine RPC-basierte API, die vom selben Google-Dienst wie Google Maps, YouTube und Google Apps-APIs gehostet wird. Sie können ihn mit den meisten Apps und Sprachen verwenden, z. B. mit einer node.js-Web-App, die auf einer virtuellen Maschine oder auf einem Java-Anwendungsserver vor Ort ausgeführt wird.

  • Google Cloud Datastore aktivieren

    Im Folgenden wird beschrieben, wie Sie die Google Cloud Datastore API aktivieren, welche Kontingente für diese API verfügbar sind und wie Sie die Abrechnung aktivieren. Der Aktivierungsprozess kann in Abhängigkeit von Ihrer bestehenden Einrichtung und Umgebung variieren.

  • Wichtige Datenspeicherkonzepte

    Der Datastore ist ein schemaloser Objektdatenspeicher, der für Ihre App eine stabile, skalierbare Speichermöglichkeit mit den folgenden Merkmalen bietet: keine geplanten Ausfallzeiten, atomarische Transaktionen, hohe Verfügbarkeit von Lese- und Schreibvorgängen, hohe Konsistenz für Lesevorgänge und Ancestor-Abfragen und Eventual Consistency für alle anderen Anfragen.
