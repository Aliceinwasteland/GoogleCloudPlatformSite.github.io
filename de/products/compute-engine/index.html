Compute Engine

Bewältigen Sie hohe Arbeitslasten auf virtuellen Maschinen, die auf der Infrastruktur von Google gehostet sind. Wählen Sie eine virtuelle Maschine, die Ihren Anforderungen entspricht, und profitieren Sie von der Leistungsfähigkeit des globalen Glasfasernetzwerks von Google.

Jetzt starten

Funktionen

Virtuelle Maschinen mit hoher Leistung

Die virtuellen Linux-Maschinen von Compute Engine bieten konsistente Leistung, Skalierbarkeit, hohe Sicherheit und Zuverlässigkeit. Es werden z. B. die Linux-Distributionen Debian und CentOS unterstützt. Sie können virtuelle Mikromaschinen bis hin zu sehr großen Instanzen wählen.

Unterstützung durch das globale Google-Netzwerk

Erstellen Sie große Rechencluster, die von der hohen, konsistenten, computerübergreifenden Bandbreite profitieren. Stellen Sie über das private, globale Glasfasernetzwerk von Google eine Verbindung zu Computern in anderen Rechenzentren und zu anderen Google-Diensten her.

Nur für die tatsächliche Nutzung zahlen

Dank der minutengenauen Abrechnung brauchen Sie nicht für Rechenzeit zu zahlen, die Sie gar nicht genutzt haben. Die in Rechnung gestellte Mindestdauer beträgt dabei 10 Minuten. Mit unseren Rabatten für eine anhaltende Nutzung erhalten Sie bei einer längerfristigen Auslastung automatisch günstigere Preise, ohne dass eine Anmeldung oder Vorabverpflichtung erforderlich wäre.

Lastenausgleich

Mittels nativer Technologie für den Lastenausgleich können Sie den eingehenden Netzwerkverkehr auf mehrere Instanzen verteilen und dadurch kostengünstig Leistung, Durchsatz und Verfügbarkeit maximieren.

Schnelle, einfache Bereitstellung

Die intuitiven Tools, einschließlich einer RESTful API, einer Befehlszeilenschnittstelle und einer webbasierten Konsole ermöglichen eine schnelle Bereitstellung großer Cluster virtueller Maschinen. Mit Tools wie RightScale und Scalr können Sie die Bereitstellung auch automatisch verwalten.

Compliance und Sicherheit

Alle auf Laufwerken in Compute Engine gespeicherten Daten werden mit dem AES-128-CBC-Algorithmus verschlüsselt. Compute Engine hat die Zertifizierungen ISO 27001, SSAE-16, SOC 1, SOC 2 und SOC 3 erhalten, was unser Engagement für die Informationssicherheit unterstreicht.

Preise

Für alle Maschinentypen werden mindestens 10 Minuten in Rechnung gestellt. Wenn Sie Ihre Instanz zum Beispiel 2 Minuten lang ausführen, wird Ihnen eine Nutzung von 10 Minuten berechnet. Nach 10 Minuten werden Instanzen im Minutentakt mit Aufrundung auf die nächsthöhere Minute abgerechnet. Wird eine Instanz beispielsweise 11,25 Minuten ausgeführt, wird Ihnen eine Nutzung von 12 Minuten berechnet.

Die Preise gelten ab 1. April 2014.

Rabatte für anhaltende Nutzung

Sobald Sie eine Instanz zu mehr als 25 % eines Abrechnungszeitraums nutzen, reduziert sich Ihr Preis. Dieser Rabatt wird automatisch angewendet, eine Anmeldung oder Vorabverpflichtung ist nicht erforderlich. Wenn Sie eine Instanz zu 100 % des Abrechnungszeitraums nutzen, erhalten Sie 30 % Nettorabatt auf Ihren bereits niedrigen Preis.

  • Kurzanleitung: Instanz erstellen und Apache starten

    In dieser Kurzanleitung wird Schritt für Schritt erläutert, wie Sie mit der Verwendung von Google Compute Engine beginnen. Anschließend wissen Sie, wie Sie eine Firewall konfigurieren, eine virtuelle Maschine oder eine persistente Root-Festplatte einrichten und einen Apache-Server für die Bereitstellung von Webseiten installieren.

  • Was ist Google Compute Engine?

    Google Compute Engine ist ein Dienst, der virtuelle Maschinen in der Google-Infrastruktur bereitstellt. Google Compute Engine bietet kostengünstige Skalierbarkeit und Leistung, sodass Sie problemlos große Rechencluster in der Google-Infrastruktur bereitstellen können. Es sind keine Vorabinvestitionen erforderlich und Sie können Tausende virtueller CPUs auf einem System ausführen, das ganz auf Schnelligkeit und hohe Leistungskonsistenz ausgerichtet ist.

  • Instanzen

    Eine Instanz ist eine virtuelle Maschine, die in der Infrastruktur von Google gehostet wird. Alle Instanzen führen Linux aus. Dabei wird ein von Google Compute Engine bereitgestelltes Standard-Image oder eine von Ihnen angepasste Version dieser Images verwendet. Bei Bedarf können Sie einen Maschinentyp wählen, der die Anzahl der CPUs, die RAM-Größe und andere Maschineneigenschaften angibt.
