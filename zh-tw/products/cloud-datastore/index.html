Cloud Datastore — Google Cloud Platform

Cloud Datastore

利用管理化的 NoSQL 無結構資料庫，儲存不具關聯性的資料。Cloud Datastore 會在您需要時，自動擴大儲存空間，此外還支援交易事務以及類似 SQL 的查詢功能。

立即試用

功能

利用類似 SQL 的查詢功能，存取各種架構的資料

您無須擔心資料模型和資料遷移事宜。Cloud Datastore 是一項無結構儲存空間服務，可讓您恣意存取各種資料，無論其基本架構為何。Cloud Datastore 提供的查詢引擎，可讓您搜尋各種屬性的資料，並在需要時加以排序。

託管的資料庫

Cloud Datastore 的管理作業是由 Google 全權負責。Google 會自動處理資料分割及複製作業，以隨時為您提供服務。

視使用者人數自動擴充

當使用者人數或需求增加時，Cloud Datastore 會自動進行擴充。如此一來，您就可以全心投入應用程式的開發作業，無須分配資料及分攤工作量。

ACID 事務處理

Cloud Datastore 利用 Optimistic Concurrency Control 技術，提供 ACID 事務處理功能。應用程式可以在單筆交易中執行多個資料儲存庫作業，無論作業的成敗與否，確保資料的完整性。

內建的備援功能

只要呼叫 API 一次，系統就會將資料自動複製到多個資料中心。另外，還提供內建的高穩定性以及耐用性。

本機上的開發工具

有了 Cloud Datastore Development Kit 之後，您只須在自家電腦上開發、測試應用程式，部署相關工作就交給 Google 負責。

隨時隨地取得所需資料

設計各種利用 App Engine 和 Compute Engine 的解決方案，並全部整合至 Cloud Datastore。您可利用 Cloud Datastore 提供的 RESTful 介面，任何部署目標都可以輕易取得資料。

價格

Google Cloud Datastore 的費用和免費級別與 App Engine 的資料儲存庫相同，詳情請參閱下表。而由於 Cloud Datastore 有一個服務元件在 Google App 上執行，因此會每小時會產生執行個體費用。

  • Google Cloud Datastore 簡介

    Google Cloud Datastore 提供一套工具和用戶端程式庫，可讓您從 Google Cloud Compute Engine 執行個體上的用戶端或其他位置，存取及管理 Google Cloud Datastore。Google Cloud Datastore 提供一種 RPC 式 API，是由代管 Google 地圖、YouTube 和 Google Apps API 的相同 Google 服務所代管。您可搭配大多數應用程式和語言 (例如，在虛擬機器中或在公司內部 Java 應用程式伺服器上執行的 node.js 網路應用程式) 使用 Google Cloud Datastore。

  • 啟用 Google Cloud Datastore

    以下說明如何啟用 Google Cloud Datastore API、這個 API 的相關配額，以及如何啟用計費功能。啟用程序可能會根據您目前的設定和使用環境，而有所不同。

  • Datastore 的重要概念

    Datastore 是一種無結構物件資料儲存庫，可為您的應用程式提供穩定、可擴充的儲存空間，並且具備以下特色：無須進行停機維護作業、支援微量交易作業、讀寫穩定性高、可確保讀取及祖系查詢，以及所有其他查詢的結果一致性。
