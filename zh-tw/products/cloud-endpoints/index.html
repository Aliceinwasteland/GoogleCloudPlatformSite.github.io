Cloud Endpoints

自行建立 RESTful 服務，讓使用者透過 iOS、Android 和 JavaScript 用戶端加以使用。自動產生用戶端程式庫，輕鬆建置前端平台。內建的功能包括拒絕服務 (denial-of-service) 攻擊防護、OAuth 2.0 支援以及用戶端金鑰管理。

立即試用

功能

將單一工具供多個用戶端使用

利用單一原始程式碼，為 Android、iOS 和網頁用戶端建立用戶端程式庫。Cloud Endpoints 只要幾個步驟即可將您的程式碼封包，以建置 API 伺服器。Google 提供 Java、Python、Go 和 PHP 專用的 Cloud Endpoint API 程式庫。請參閱我們的說明文件瞭解 Cloud Endpoint，或者試用網路應用程式範例 Tic Tac Toe。

擴充 App Engine 基礎架構

現在，您的行動裝置也可取得 App Engine 提供的所有工具和程式碼。只須透過 App Engine 後端平台即可存取 Datastore、Cloud Storage 和 Task Queues，而不須進行任何設定。請按照我們的範例，整合 OAuth 2.0 驗證功能。

無須維護用戶端和伺服器

由於 App Engine 支援 Cloud Endpoints，因此您不需要維護任何伺服器、不用分攤工作負載，也不必害怕進行繁雜的擴充作業。與 App Engine 和我們的其他雲端服務一樣，用多少付多少。

具彈性的用戶端整合

標註您的伺服器 API，即可讓系統自動建置用戶端程式庫。接下來為 Android 和 iOS 建立用戶端程式庫。只要少數幾個 JavaScript 用戶端程式庫，即可讓標準網路用戶正常執行。所有用戶端均採用相似的 API，和相同的後端平台，可協助您節省開發時間。

價格

在 App Engine 和 Compute Engine 上執行的應用程式，可以免費使用 Cloud Endpoints。

參考資源

  • 建立 Endpoints API

    Endpoints API 是一項遠端程序呼叫 (RPC) 服務，可為外部用戶端提供各種遠端方法。每個 Endpoints API 均包含一個 RPC 服務類別，這個類別包含 ProtoRPC remote.Service 類別以及一或多個方法。

  • 產生用戶端程式庫

    建立好端點之後，即可為 Android 和 iOS 產生用戶端程式庫。

  • 在 iOS 用戶端使用端點

    如要在 iOS 應用程式使用端點，請您進行以下動作：編譯用戶端程式庫產生器，然後產生您的程式庫。然後將所需檔案加到 iOS 專案。接著建立服務物件，以用來發出要求。呼叫端點找到的 API。最後視情況，為通過驗證的呼叫增加支援。
