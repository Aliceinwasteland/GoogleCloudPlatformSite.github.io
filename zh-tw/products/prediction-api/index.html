Google Prediction API - Predictive Analysis & Prediction Algorithms — Google Cloud Platform

Prediction API

利用 Google 的機器學習演算法，並搭配熟悉的 RESTful 介面，即可分析及預測結果。

立即試用

功能

讓資料發揮用處

利用歷史資料預測未來趨勢。利用 Prediction API 傳送訊息、偵測垃圾郵件，以及向使用者推薦產品。Google 完善的基礎架構會協助您分析資料。

既快速又可靠

Google 會利用 Cloud Storage，將您的資料複製到數個資料中心。大部分的預測查詢所需的處理時間不會超過 200 毫秒，為您提供更好的效率。如需改善效率的提示，請參閱這份說明文件。

雲端整合

Google 的每一項服務皆可與彼此搭配使用。您可以利用 Google BigQuery 尋找資料，或者從 Cloud Storage 提取最多 2.5 GB 的資料集，來建立可預測的模型。您可以直接透過 Google App Engine 使用 Prediction API。

實用的開發工具

Prediction API 可與 App Engine 搭配使用，而且很多像 Python、JavaScript 和 .NET 此類主流語言的程式庫皆有提供 RESTful API。如需完整清單，請參閱這份說明文件。

範例和支援

請參考我們的應用程式範例，深入瞭解 Prediction API。請參閱我們的示範以及精選的應用程式，激盪出更多創意。

具彈性的計費方式

前 6 個月免費，但有使用次數限制。而針對 Prediction API 的定價，Google 推出 99.9% 可用的服務水準協議。與其他雲端產品相同，用多少付多少。

價格

免費配額

前 6 個月免費使用，但每一項 Google Cloud Console 專案設有以下使用限制。即使您啟用計費功能，前 6 個月仍為免費使用。

  • 使用限制：
    • 預測：每天 100 次預測
    • 代管模型預測：代管模型的用量限制，為所有模型每位使用者每天 100 次預測。
    • 培訓：每天 5 MB 的培訓用量
    • 串流更新：每天 100 次串流更新
  • 有效期內保證額度：20,000 次預測。
  • 期限：6 個月的免費使用期截止後，請前往 Google Cloud Console 為專案啟動 Google Prediction。

請注意，部分 Prediction API 使用量受限於 Google Cloud Storage，在這種情況下，Prediction API 會有不同的計費方式。

付費使用

付費使用適用 99.9% 正常可用的服務水準協議。如要進一步瞭解 SLA，請參閱這裡。

請到 Google Cloud Console 監控您的使用量。免費使用期截止後，如果您未啟用專案的計費功能，就無法繼續使用。超過免費配額時，即適用於以下價格和限制：

  • 基本費用：每項 Google Cloud Console 專案每月 $10。
  • 使用費用：
    • 預測：
      • 每月每 10,000 次預測 $0.00
      • 每月每 10,001 次預測以上：自 10,001 次預測起，每 1,000 次預測 $0.50
    • 培訓：
      • 每 1 MB 的培訓用量 $0.002 (每個資料集的大小上限為 2.5 GB)
      • 0-10,000 次串流更新：$0.00
      • 10,001 次串流更新以上：自 10,001 次串流更新啟，每 1,000 次串流更新 $0.05。
  • 使用限制：
    • 如果您一天會進行 40,000 次以上的預測，請與我們聯絡。
    • Google Prediction 預設的預測次數上限額度，為您模型中的每項專案每天 2,000,000 次預測。
    • 代管模型的用量限制，為所有展示模型每位使用者每天 100 次預測。開發人員可以為自行建立的模型，設定更高的付費使用額度。詳情請參閱相關模型的說明文件。

請注意：以上費用「不」包括任何 Google Cloud Storage 儲存培訓資料所產生的費用。請參閱 Google Cloud Storage 價目表，判斷您可能需要支付的額外費用。

參考資源

  • 什麼是 Google Prediction API？

    Google 的雲端機器學習工具，可以分析您的資料，然後將以下的功能加到應用程式中：客戶情緒分析、垃圾郵件偵測、郵件寄送判斷、加碼銷促商機分析、文件和電子郵件分類等其他功能。

  • 預測購買

    假設有一個網站專門銷售啤酒、紅酒和乳酪，而您想根據訪客的消費紀錄，預測對方是否有意願購買紅酒。在這種情況下，我們會建立含有 3 種數據的培訓資料：客戶曾經購買乳酪的次數；如果該客戶曾經購買紅酒，值為 1，如果沒有，則值為 0；以及，客戶曾經購買啤酒的次數。

  • 建立情緒分析模型

    本文件說明如何利用 Google Prediction API 建立基本的情緒分析模型。情緒分析模型可用於分析文字字串，並加以分類到您指定的標籤。舉例來說，您可以分析一則推文，判斷對方的心情是好是壞，或者分析一封電子郵件，判斷對方情緒是快樂、受挫還是悲傷。
