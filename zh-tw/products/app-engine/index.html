App Engine

利用內建的服務，在全面管理化的 PaaS (Platform-as-a-Service，平台即服務) 服務上執行應用程式，可提高您的工作效率。您只須下載 SDK，即可開始建置應用程式。

立即試用

功能

主流的語言和架構

利用以下主流的程式設計語言撰寫應用程式：Python、Java、PHP 和 Go，並且使用既有的架構 (例如 Django、Flask、Spring 和 webapp2)，在自己的電腦上使用各種語言專用的 SDK 開發應用程式。將您的應用程式與 Compute Engine 搭配使用，即可整合其他熟悉的技術，例如 Node.js、C++、Scala、Hadoop、MongoDB、Redis 等。

專心設計程式碼

讓 Google 負責管理資料庫、設定資料庫、分割資料以及平衡負載。您可利用流量分割 (Traffic Splitting) 技術，測試兩種不同的應用程式版本。多租戶架構支援可讓您劃分應用程式資料。

多種儲存空間方案

選擇您需要的儲存空間方案：利用 Cloud SQL 的傳統 MySQL 資料庫、無結構 NoSQL 資料儲存庫，或利用 Cloud Storage 的物件儲存空間。

功能強大的內建服務

App Engine 不只可協助提高工作效率，還可讓您有更多時間撰寫全新的程式碼。管理化服務 (例如 Task Queues、Memcache 和 Users API) 則可協助您建置各種應用程式。

熟悉的開發工具

利用您熟稔的工具開發應用程式，包括 Eclipse、IntellIJ、Maven、Git、Jenkins、PyCharm 等。App Engine SDK 可讓您在模擬環境中測試應用程式，然後利用簡易的命令行工具或桌面啟動器部署您的應用程式。

部署到 Google 大規模的基礎架構上

我們的平台內建了幾項享譽全球的網路服務。您每天可以擴大處理最多 70 億個要求，而系統會在流量下降時，自動調降處理能力。

個案研究

許多開發人員皆是在 App Engine 上建置各種應用程式，其中包括可視需要擴充的網路應用程式、行動應用程式、遊戲、企業應用程式等。

參考資源

  • 什麼是 Google App Engine？

    Google App Engine 可讓您在 Google 的基礎架構上執行網路應用程式。App Engine 應用程式的建置簡單、維護容易，而且可以因應流量和資料儲存量輕鬆擴充。使用 App Engine 就不需要維護伺服器：您只要上傳應用程式，就可以馬上為您的使用者提供服務。

  • 為何選用 App Engine？

    您可以利用 Google App Engine，在可視需要擴充的系統上建置網路應用程式，該系統具備各種 Google 應用程式。App Engine 應用程式的建置簡單、維護容易，而且可以因應流量和資料儲存量輕鬆擴充。使用 App Engine 就不需要維護伺服器：您只要上傳應用程式，就可以馬上為您的使用者提供服務。

  • App Engine 功能

    這個網頁概述 App Engine 的各種功能。部分功能可供所有或少數執行階段語言使用。雖然每次實際執行時，功能一般都是固定不變的，不過還是會有例外。其中的表格除了概述每項功能之外，並有提供功能所支援語言的連結，點選連結即可開啟該語言的專屬網頁，以獲得更多資訊。
