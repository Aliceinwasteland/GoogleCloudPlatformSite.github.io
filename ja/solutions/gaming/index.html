Google Cloud Endpoints

Google Cloud Storage

Google のストレージ インフラストラクチャにゲームのメディア ファイルやサイズの大きなファイルを保存して、アクセスや管理を行えます。データ保存の信頼性と冗長性を確保しながら、クライアントの端末に直接データを配信できます。

詳細

ソリューション

サイズの大きなメディア ファイルを配信

高解像度の画像とあらゆるサイズでダウンロードすることに対するユーザーのニーズは増えています。新しいコンテンツを提供し続けることによって継続的に利用するユーザーを確保し、アプリケーションへの関心を維持することが求められています。Google Cloud Storage では、シンプルな REST ベースのインターフェースを使用してサイズの大きなファイルをクラウドに保存でき、Google のアプリケーションと同じ高速ネットワークを使用しています。

ユーザーの関心を分析

ユーザーの行動やゲームのパフォーマンスを把握することは、アプリケーションの価値を高める鍵です。しかし、このためには、ログに記録するだけでなく、強力な分析が必要です。Google BigQuery は、アプリケーションのこうした課題に対応するため、Google で使われているのと同じテクノロジーを公開して、テラバイト単位のデータに対するリアルタイムのアドホック分析を実行し、ダッシュボードにフィードします。また、App Engine の MapReduce を使用して、膨大なデータセットを並行処理して非リアルタイム分析を実行することもできます。

複数の端末間でゲームのステータスを同期

ゲームのステータスを App Engine データストアや Google Cloud Storage に保管することで、ユーザーは所有するすべての端末でゲームのステータスを見ることができるようになり、開発者はより詳細な分析情報を得ることができるようになります。

携帯端末との簡単な統合

Google Cloud Endpoints を使用することにより、サーバーのロジックをウェブ API として自動的に公開し、iOS/Android 用に強く型付けされたクライアント ライブラリ、標準的なウェブ アプリ用に JavaScript を生成できます。

Google Cloud Platform ではさまざまなゲームをビルド可能

Google Cloud Platform では、Android や iOS などのモバイル プラットフォーム用と、Chrome OS や Facebook などのウェブ プラットフォーム用にゲームをビルドできます。ターン制のゲームからソーシャル ゲームまでさまざまなタイプのゲームをビルドすることもできます。

Google でのゲームの配信と収益化

Google AdSense と Google アプリ内ペイメントを使用して、ゲームを収益化できます。Adsense では、ゲームの外観と使用感に合致するターゲット広告を表示できます。アプリ内ペイメントでは、取引あたり 5% のフラット料金で仮想グッズを販売できます。ウェブ用ゲームの場合は Chrome ウェブストアを使用して、モバイル用ゲームの場合は Google Play を使用してゲームを配信できます。
