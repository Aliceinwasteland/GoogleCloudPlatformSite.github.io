Google Cloud Platform で Hadoop を利用してすばやく拡大

Apache Hadoop ソフトウェア ライブラリは、シンプルなプログラミング モデルを使用して大量のデータセットを複数のコンピュータ クラスタ間で分散処理できるようにするフレームワークです。1 つのサーバーから数千のマシンにまで拡大することが可能で、各マシンがローカル計算とストレージの機能を提供します。ハードウェアによって高可用性を実現するのではなく、ライブラリ自体がアプリケーション レイヤでの障害を検出して処理するため、障害が発生しがちなコンピュータ クラスタより上位で高可用性サービスを実現できます。

Google Cloud Platform で Hadoop を利用するメリット:

高速な起動

Compute Engine の仮想マシンの起動時間は数秒です。

高速 I/O

MapR は、Compute Engine　とHadoopを用いて MinuteSort の新記録を樹立しました。

スケールへの対応

コストを気にかけることなく、計算処理により多くの CPU を投じることができます。課金は分単位で行われるため、規模や速度に合わせた最適化が可能です。

Apache Hadoop の使用

永続的な Hadoop クラスタの構成方法についてはチュートリアルをご確認ください。このサンプルでは、MapReduce タスクを利用して Google Cloud Storage と HDFS 間で MapReduce の入出力のコピーを並列処理しています。

YouVersion アーキテクト Josh Turmel 氏

MapR Technologies

MapR では Google Compute Engine を活用し、150 億 100 バイトのレコード（合計 1 兆 5,000 億バイト）を 60 秒でソートするという MinuteSort の新記録を樹立しました。データ ソート処理のワールド カップとも呼ばれるこのベンチマークでは、ディスクの最初と最後でのデータのソート速度を実証します。前回の MinuteSort 記録はカスタム ハードウェアによって達成されましたが、MapR は今回、新記録を Google Compute Engine、Hadoop MapReduce、Apache Hadoop MapR ディストリビューションよって樹立しました。
