BigQuery

クラウド内のビッグ データを BigQuery で解析できます。マルチテラバイトのデータセットに対し、SQL に似たクエリを数秒で簡単に実行できます。使いやすくスケーラブルな BigQuery により、ユーザのデータをリアルタイムで解析できます。

今すぐ試す

特長

すべてバックグラウンドで実行可能

クエリをバックグラウンドで非同期で実行でき、ステータスをポーリングできます。Cloud Platform の他のリソースにアクセスできる Google Cloud Console で、ジョブやクエリの履歴にアクセスできます。

データのインポートが簡単

Google Cloud Storage を使ってバルクでデータをロードすることもできますが、ストリームで１秒間に1000行単位でロードすることもできます。また、リアルタイムでデータを流し込むことが可能です。ハイボリュームのロギング、トランザクションの記録、そして、高速分析が可能です。

手頃な料金でビッグデータを処理

Google が「ビッグ データ」と言うときはまさに大容量のデータを指します。毎月、最初の 100 GB のデータの処理については課金されません。Google がお客様の代わりに数千テラバイトのデータを保存し、クエリを実行します。Google の料金体系はシンプルで明快です。ご利用形態を決めるのはお客様です。お支払いいただくのは、必要なだけのストレージと実行したクエリに対してのみとなります。

適切なインターフェース

管理用と開発用のインターフェースが別になっているため、必要なツールに確実にアクセスできます。Google API コンソールを使用して、プロジェクト レベルとデータセット レベルの両方でアクセスを制御できます。BigQuery を使用することにより、あらゆる規模のチームが理想どおりに作業しやすくなります。

料金

BigQuery は、カラム型データ構造を使用しており、課金方法は、処理されたカラム分のみとなり、テーブル全体が課金単位とはなりません。月に100GBまでのデータ処理は無料です。

この料金は 2014 年 4 月 1 日より適用されます。

リソース

  • BigQuery とは

    適切なハードウェアとインフラストラクチャを用意せずに大規模なデータセットに対してクエリを実行すると、時間や費用がかかる場合があります。Google BigQuery は、Google のインフラストラクチャの処理力を使用して書き込み専用テーブルに対して SQL に似たクエリを超高速で実行することで、この問題を解決しています。データを BigQuery に読み込んだら、後の処理は Google にお任せください。

  • BigQuery へのデータの読み込み

    データに対するクエリを実行するには、その前にデータを BigQuery に読み込む必要があります。ジョブを使用してデータを一括で読み込んだり、個別のレコードをストリーミングしたりできます。

  • データのクエリ

    クエリは、BigQuery の SQL 言語で記述します。BigQuery は、同期クエリと非同期クエリの両方のクエリ方法をサポートしています。どちらの方法もジョブによって処理されますが、「同期」方法では、結果を返す前にジョブが終了するまで待機するタイムアウト値が公開されます。
