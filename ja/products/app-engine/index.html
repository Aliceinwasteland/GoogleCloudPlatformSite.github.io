App Engine

生産性を高めるビルトイン サービスを使用した完全管理型の PaaS（Platform-as-a-Service: サービスとしてのプラットフォーム）でアプリケーションを実行できます。SDK をダウンロードするだけで、すぐにビルドを開始できます。

今すぐ試す

特長

一般的な言語とフレームワークへの対応

Python、Java、PHP、Go などのよく使われるプログラミング言語を使用してアプリケーションを記述できます。また、Django、Flask、Spring、webapp2 などの既存のフレームワークに対応しています。言語固有の SDK を使用してローカルで開発できます。アプリケーションを Compute Engine と組み合わせれば、Node.js、C++、Scala、Hadoop、MongoDB、Redis などのよく使われる技術を統合することも可能となります。

ユーザがコードに集中できるように

データベース管理、サーバー設定、シャーディング、負荷分散は Google にお任せください。トラフィック分割を使用することで、アプリのさまざまなライブ バージョンの A/B テストを実施できます。マルチテナンシー サポートにより、アプリケーション データの区分化も可能です。

複数のストレージ オプションを用意

Cloud SQL を使用した従来の MySQL データベース、スキーマレス NoSQL データストア、Cloud Storage を使用したオブジェクト ストレージの中からニーズに応じたストレージ オプションを選択できます。

強力なビルトイン サービス

App Engine は、ボイラープレート コードを記述する必要性を排除することにより、高い生産性を実現しています。タスク キュー、Memcache、Users API などのマネージドサービスにより、どんなアプリケーションでも作成できます。

一般的な開発ツール

Eclipse、IntellIJ、Maven、Git、Jenkins、PyCharm などのよく使われるツールを使用できます。App Engine SDK では、シミュレートされたローカル環境でアプリケーションをテストしてから、簡単なコマンドライン ツールやデスクトップ ランチャーを使用してアプリを展開できます。

Google 規模での展開

Google のプラットフォームには、世界で最もよく使われているウェブ サービスが複数ビルトインされています。1 日あたり最大 70 億件のリクエストまで拡張でき、トラフィックの減少時には自動的にスケール ダウンさせることが可能です。

  • Google App Engine について

    Google App Engine を使用すると、作成したウェブ アプリケーションを Google のインフラストラクチャ上で実行できます。App Engine アプリケーションはビルドや維持管理が簡単で、トラフィックやデータストレージの増大に合わせて容易に拡張できます。App Engine では、サーバーを維持管理する必要もありません。アプリケーションをアップロードするだけで、ユーザーに配信する準備が整います。

  • App Engine を使用するメリット
