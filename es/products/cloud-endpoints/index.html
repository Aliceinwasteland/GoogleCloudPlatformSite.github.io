Cloud Endpoints — Google Cloud Platform

Cloud Endpoints

Crea servicios REST y haz que sean accesibles desde clientes iOS, Android y JavaScript. Genera automáticamente bibliotecas cliente para que la interfaz sea fácil de configurar. Las funciones integradas incluyen protección contra ataques de denegación de servicio, compatibilidad con OAuth 2.0 y administración de claves de clientes.

Pruébalo ahora

Funciones

Una herramienta, varios clientes

Crea bibliotecas cliente para Android, iOS y clientes basados en web desde un solo origen. Cloud Endpoints empaqueta el código para crear un servidor de API en unos pocos pasos. Las bibliotecas de la API de Cloud Endpoints están disponibles en Java, Python, Go y PHP. Obtén más información sobre Cloud Endpoints en la documentación o lánzate a la piscina y prueba la aplicación web de ejemplo Tic Tac Toe.

Ampliación de la infraestructura de App Engine

Ahora, todas las herramientas y bibliotecas disponibles en App Engine también están disponibles en tus dispositivos móviles. Accede al almacén de datos, a Cloud Storage y a las colas de tareas mediante el backend de App Engine sin trabajo adicional. Para integrar rápidamente la autenticación OAuth 2.0, sigue estos ejemplos.

Cliente-servidor de bajo mantenimiento

Como Cloud Endpoints está respaldado por App Engine, no tienes que realizar el mantenimiento de ningún servidor, no tienes que preocuparte por la distribución de la carga y el ajuste de la escala es rápido y sencillo. Como App Engine y los otros servicios de Cloud, solo pagas por lo que utilizas.

Integración flexible de clientes

Anota la API del servidor y, a continuación, crea las bibliotecas cliente automáticamente. Las bibliotecas cliente se crean para Android e iOS. Pon en marcha clientes web estándar con una biblioteca cliente JavaScript mínima. Todos los clientes utilizan API parecidas y el mismo backend, lo que reduce el tiempo de desarrollo.

Precios

Cloud Endpoints está disponible gratis para las aplicaciones que se ejecuten en App Engine y Compute Engine.

Documentación y recursos

Recursos

  • Creación de una API de Endpoints

    Una API de Endpoints es un servicio de llamada remota de procedimiento (RPC) que proporciona métodos remotos accesibles desde clientes externos. Cada API de Endpoints consta de una clase de servicio RPC que tiene como subclases la clase remote.Service de ProtoRPC y uno o más métodos.

  • Generación de bibliotecas cliente

    Después de acabar de crear los puntos de acceso, puedes generar bibliotecas cliente para Android e iOS.

  • Uso de Endpoints en un cliente iOS

    Para utilizar Endpoints en una aplicación iOS, tienes que hacer lo siguiente: compilar el generador de bibliotecas cliente y generar la biblioteca; añadir los archivos necesarios al proyecto de iOS; crear el objeto de servicio que se utilizará para realizar solicitudes; llamar a la API expuesta por el punto de acceso. Opcionalmente, puedes añadir compatibilidad con llamadas autenticadas.
