Administración de aplicaciones complejas en la nube

Desarrollo de aplicaciones para el mundo real en Google Cloud Platform

La mayoría de las aplicaciones para empresas de hoy en día, y cada vez más, necesitan un nivel de complejidad al que es difícil que un solo servicio de alojamiento dé respuesta. Una aplicación puede incluir varios servicios lógicos, diferentes lenguajes de programación, diferentes tipos de necesidades de almacenamiento de datos y una serie de requisitos de tiempo de ejecución. Además, los equipos suelen estar formados por desarrolladores con áreas de especialización y con niveles de experiencia muy diferentes. Los requisitos cambian a un ritmo que a menudo conduce a un nivel de complejidad cuya administración supone un reto hasta para los equipos más grandes.

Presentación

Google Cloud Platform ha evolucionado para proporcionar un completo conjunto de herramientas para administrar hasta los requisitos más complejos de las aplicaciones. Tanto si se utiliza en una aplicación completamente administrada como si se aplica a máquinas virtuales personalizadas, ofrece a los desarrolladores la flexibilidad necesaria para equilibrar la carga administrativa con las exigencias técnicas. En Google Cloud Platform, las aplicaciones se pueden desarrollar fácilmente como componentes independientes con requisitos técnicos y empresariales muy diferentes. Estos componentes pueden tener su propia configuración de rendimiento, su propio presupuesto, su propia autorización y su propio control de versiones, a la vez que comparten un conjunto común de recursos. Los recursos compartidos permiten obtener una integración más estrecha que las aplicaciones individuales que se comunican a través de API tradicionales. Además, los componentes se benefician de un modelo integrado de seguridad y de la administración centralizada propia de un proyecto en la nube.[1] El desarrollo en Cloud Platform proporciona una serie de herramientas para administrar la creciente complejidad de las aplicaciones.

Tanto si quieres crear una aplicación desde cero como si tu intención es trasladar una aplicación personalizada ya existente[2] a la nube, Google Cloud Platform puede proporcionarte el nivel de rendimiento y de funciones adecuado para tu negocio.

En esta solución, te mostraremos las herramientas adecuadas para:

  • diferentes niveles de conocimientos de los desarrolladores,
  • diferentes requisitos de almacenamiento de datos,
  • requisitos interactivos y de procesamiento por lotes,
  • motores de análisis personalizados,
  • componentes empresariales separados por lógica.

Ámbito y situación

Este artículo está diseñado para los arquitectos de soluciones y para los directores tecnológicos interesados en administrar la implementación y en el escalado de una aplicación de gran tamaño en Google Cloud Platform. En el ejemplo que se ofrece en el artículo se muestra cómo gestionar muchas de las complejidades a las que se hace frente durante la implementación de aplicaciones de gran tamaño. Proporcionamos una introducción a las herramientas de la plataforma y ofrecemos enlaces a datos más detallados siempre que resulta pertinente. El ejemplo que se utiliza en este artículo se ha mantenido a propósito en niveles de cierta sencillez para centrarse en resolver las complejidades habituales en vez de profundizar en funciones específicas de la aplicación.

Para ilustrar cómo Google Cloud Platform te permite administrar aplicaciones complejas en la nube, describiremos paso a paso una situación del mundo real en la que se crea una comunidad para compartir vídeos. En esta situación, los usuarios podrán subir vídeos desde un teléfono móvil. El contenido se transcodificará para reproducirlo en varios formatos y, finalmente, se presentará en un sitio de la comunidad web para verlo y comentarlo. La aplicación de demostración para la situación incluye los siguientes componentes:

  • aplicación de Android para subir vídeos y backend móvil,
  • canalización de transcodificación de vídeo,
  • sitio web de la comunidad para ver vídeos y añadir comentarlos,
  • análisis de comentarios a través de Hadoop.

Se tratarán los siguientes retos técnicos:

  • varios lenguajes de backend,
  • varios equipos o desarrolladores independientes,
  • mezcla de aplicaciones administradas y binarios personalizados,
  • varios requisitos de almacenamiento de datos,
  • escalado dinámico para el procesamiento por lotes.

También puedes ver una demostración en directo que ofrecimos en Google I/O 2013.

En la solución que proponemos se utilizan los siguientes productos de Cloud Platform:

  • App Engine
  • Colas de tareas de App Engine

  • Cloud Endpoints para móviles
  • Módulos de App Engine

En la solución también se utilizan siguientes herramientas de terceros y de código abierto (no avaladas por Google):

  • Apache Hadoop
  • FFmpeg

Administración de la implementación y escalado de los servicios web

Para realizar la implementación y las tareas de mantenimiento de una propiedad web estándar, hay gran cantidad de gastos generales asociados a la administración de la red, de los sistemas y de los servicios subyacentes. App Engine proporciona una plataforma de aplicaciones totalmente administrada para paliar estas preocupaciones. Gestiona automáticamente el escalado de recursos para aumentarlos o reducirlos según la demanda. El control de versiones es otra función de gran utilidad que permite a los desarrolladores gestionar fácilmente la retirada de versiones, dividir el tráfico de clientes entre varias versiones y llevar a cabo un lanzamiento lento de las nuevas versiones. En el caso de las cargas de trabajo web, App Engine ayuda a un desarrollador a centrarse por completo en crear las mejores aplicaciones.

La parte web del sitio para compartir vídeos se ha creado en App Engine para aprovechar estas ventajas únicas. Este componente se ha escrito en uno de los lenguajes más populares de la Web, PHP. La compatibilidad con PHP en App Engine aporta todas las ventajas del escalado automático, del control de versiones y del rendimiento, todo ello combinado con el rápido desarrollo que viene posibilitado por este lenguaje creado para la Web. La mayoría de las aplicaciones en PHP ya existentes se ejecutarán con cambios mínimos en el entorno de App Engine.

Puesto que muchos equipos de desarrollo web están familiarizados con MySQL, hemos optado por utilizar una tecnología de bases de datos que probablemente dominen: Google Cloud SQL. Esta tecnología proporciona compatibilidad con las funciones de MySQL y con el protocolo de SQL, de modo que la transición de las cargas de trabajo ya existentes es muy sencilla. Además, Google se encarga de gestionar en segundo plano la complejidad de la alta disponibilidad y de las copias de seguridad. En el ejemplo, Cloud SQL se utiliza para el almacenamiento principal de datos de usuario y de metadatos de vídeo.

Para el almacenamiento de archivos de vídeo y de miniaturas, hemos optado por Google Cloud Storage, que admite archivos de gran tamaño y hace envíos directos a nuestros clientes de la aplicación. También proporciona una caché perimetral opcional para la distribución general de nuestro contenido de vídeo. Para las subidas, Cloud Storage ofrece URL de duración limitada que se pueden editar para permitir que el cliente suba archivos directamente. Así se reduce la necesidad de gastar valiosos recursos de aplicaciones para transmitir los archivos al almacenamiento.

Para la actividad del usuario en el sitio, es decir, para los comentarios, hemos previsto un crecimiento a una escala considerablemente mayor y hemos optado por utilizar la base de datos NoSQL de App Engine, el almacén de datos de replicación con alta disponibilidad (HRD). HRD proporciona un nivel de escalabilidad que otras soluciones puramente SQL no pueden igualar y es una buena opción para el acceso a datos de gran volumen. Esta elección también nos abrió las puertas al uso de la recién publicada función Cloud Datastore, que proporciona acceso a la actividad de nuestros usuarios en Cloud Platform y en el exterior.

Administración de canalizaciones de procesamiento por lotes

Hoy en día, la mayoría de los sitios web grandes necesita cierto nivel de procesamiento por lotes. Las cargas de trabajo pueden incluir el procesamiento del contenido subido, la actualización de los datos de usuario, la administración de notificaciones externas o el análisis de datos. Sea cual sea la demanda empresarial, las canalizaciones de procesamiento por lotes son, por naturaleza, difíciles de crear, de escalar y de mantener. Google Cloud Platform proporciona las herramientas y los servicios para simplificar la tarea de crear una canalización de procesamiento escalable y fiable. Tanto si se trata de un entorno de procesamiento programable como si se trata de una gama de opciones de almacenamiento, la plataforma está preparada para el procesamiento por lotes distribuido.

En la situación que planteamos se utiliza una canalización de procesamiento por lotes para administrar el contenido de vídeo subido a la aplicación. La mayoría de los sitios para compartir vídeos utilizan el procesamiento por lotes para el contenido subido y para todas las tareas que implica, desde transcodificarlo hasta añadir marcas de agua o administrar los derechos digitales. La demanda de estos servicios tiende a ser muy variable en función de la subida esporádica de contenido o de un comportamiento de consumo más estable. Además, los diversos componentes de una canalización de vídeo tienden a requerir aplicaciones o código personalizados que no son adecuados para un entorno puramente PaaS (Plataforma como servicio). Ambos factores convierten Compute Engine en una excelente opción. El escalado de un centro de datos o de un alojamiento administrado estándar suele requerir la compra de capacidad por anticipado o el pago de bloques de una hora. Google Compute Engine ofrece una puesta en marcha de extrema rapidez para las instancias (en la mayoría de los casos, en segundos) y una retirada rápida, por lo que es una excelente opción para este tipo de carga de trabajo. Además, Compute Engine proporciona un rendimiento muy coherente durante el escalado.

Para nuestro entorno, utilizamos una combinación de App Engine y de Compute Engine para poner en marcha un servicio de transcodificación de vídeo escalable a petición. Usamos un motor de organización escrito en Python en App Engine que nos permite controlar el comportamiento de la instancia de Compute Engine mediante la API de JSON disponible y la actividad coordinada de transcodificación mediante las colas de tareas. La configuración nos permite asignar de manera dinámica la capacidad para responder a la demanda y a la carga existente. Además, una secuencia de comandos periódica comprueba la carga en el entorno y reduce la capacidad cuando ya no es necesaria. Así se proporciona un considerable beneficio económico, gracias al reciente cambio en Compute Engine hacia la facturación por fracciones de hora. Las colas de tareas proporcionan una forma de aceptar, de almacenar en búfer y de distribuir solicitudes entre las diferentes instancias de transcodificación de vídeo.

Administración de implementaciones rápidas para móviles

Google Cloud Platform ofrece un conjunto de herramientas para permitir el desarrollo rápido de aplicaciones para móviles. Al combinar Cloud Endpoints y App Engine, se obtiene una plataforma para mostrar la tecnología de servidor backend como una API para el consumo externo. Sin Cloud Endpoints, añadir una aplicación para móviles a un sitio web ya existente, por lo general, requiere cierto nivel de planificación para tener una API flexible y autenticación segura. Endpoints gestiona la autenticación y la generación de API de forma automática en nombre de los desarrolladores, que así quedan libres para crear las funciones de la aplicación. Además, el uso de App Engine simplifica el escalado y las tareas continuas de mantenimiento para el servidor de la API.

Para añadir una aplicación para móviles en nuestro sitio para compartir vídeos, el servidor de la API se ha creado en App Engine. También hemos utilizado Google Cloud Endpoints para facilitar la autenticación y el desarrollo rápido de la API. Además, hemos decidido implementar el backend en Java para ilustrar la flexibilidad de opciones de lenguaje entre los diferentes componentes empresariales. Nuestro backend móvil facilita la subida de contenido de vídeo desde la aplicación para móviles a Cloud Storage, la creación de metadatos en Cloud SQL y la integración en el servicio de transcodificación a través de colas de tareas.

La aplicación de vídeo para móviles se ha escrito en Android con las bibliotecas de Cloud Endpoints generadas para integrar rápidamente la autenticación y la subida de vídeos. Además, los vídeos se suben directamente a Cloud Storage para evitar toda latencia o todo procesamiento adicional en el servidor de la API de App Engine. La autenticación se gestiona a la perfección a través de Cloud Endpoints y el permiso se delega al segmento de Cloud Storage para almacenar el contenido de vídeo. La delegación de permisos es una opción del segmento que permite a los usuarios de la aplicación subir directamente los archivos. Al combinar la autenticación simplificada y la generación de API, se ha podido desarrollar rápidamente una solución completa de recursos multimedia para móviles.

Administración del análisis de datos de usuario

El análisis de la actividad de usuario en las aplicaciones puede dar lugar a una explosión en el volumen de datos, junto con un aumento en el uso de los sistemas y del software para recuperar resultados importantes. Debido a la naturaleza esporádica de la mayoría de los análisis, los entornos aprovisionados pueden cruzarse de brazos durante periodos de tiempo prolongados. Sin embargo, al realizar análisis, estos sistemas pueden consumir grandes cantidades de recursos de procesamiento y de almacenamiento a fin de completar la tarea en plazos razonables. Estas características lo convierten en una buena opción para Compute Engine. La capacidad de poner en marcha rápidamente las instancias para responder a una carga de trabajo cada vez mayor y de reducir la escala de las instancias cuando no haya actividad permite que las tareas de análisis pequeñas y grandes se inicien a petición. Además, la disponibilidad de capacidad para expandirse evita embotellamientos asociados con entornos habituales de procesamiento de datos.

Para procesar el volumen previsto de comentarios en nuestra aplicación de vídeo, hemos optado por utilizar una instalación de Hadoop ejecutada en Compute Engine. Una vez más, la API de Compute Engine hace que sea fácil iniciar una agrupación de Hadoop de diferentes tamaños para el procesamiento de datos. En nuestro caso, extraemos datos de comentarios de Cloud Datastore asociado a nuestra aplicación y hacemos un cálculo simple sobre la opinión expresada en los comentarios asociados a un vídeo. Los datos de Cloud Datastore se comparten automáticamente entre las instancias de App Engine y de Compute Engine pertenecientes al mismo proyecto. El resultado final del cálculo de las opiniones totales negativas o positivas se vuelve a escribir en una entidad de Datastore asociada al vídeo.

Google ofrece varias alternativas para el análisis de datos. En concreto, para los análisis de opiniones, la Google Prediction API proporciona una interfaz REST a un entorno de obtención de información automático que, entre otras opciones, llevará a cabo análisis de las opiniones en datos de usuario. Para el análisis de datos más generales, Google BigQuery proporciona un motor para procesar terabytes de datos sin el gasto general del desarrollo personalizado. Otra buena alternativa al procesamiento personalizable de Datastore es MapReduce de App Engine.

Administración de la complejidad

Dada la variedad de servicios lógicos que hay en nuestra aplicación, hemos usado en gran medida varias funciones clave de Google Cloud Platform. Juntas, proporcionan una base férrea para admitir aplicaciones empresariales complejas en la nube.

Cloud Endpoints

Cloud Endpoints es una colección de herramientas y de servicios para generar API REST y bibliotecas cliente desde una aplicación de App Engine para simplificar el acceso tanto de clientes web como de clientes móviles. La combinación permite un rápido lanzamiento de un servidor de la API para aplicaciones para Android. Cloud Endpoints simplifica la comunicación con una aplicación de App Engine que posibilita las ventajas de los servicios disponibles, por ejemplo: Cloud Datastore, Cloud Storage, correo, colas de tareas, etc. En segundo plano, Cloud Endpoints convierte las llamadas de la API REST en invocación de métodos en la aplicación backend y protege cada llamada con autenticación OAuth 2.0. En la actualidad, los entornos de desarrollo existentes, por ejemplo, Eclipse, Android Studio y Apache Maven, ofrecen funciones para generar una biblioteca cliente desde dentro de un proyecto. La tecnología de Cloud Endpoints elimina el gasto general del desarrollo de APIs y permite a los desarrolladores centrarse en lo que de verdad importa, su producto.

Para empezar a utilizar Google Cloud Endpoints, echa un vistazo a la sesión From Nothing to Nirvana in Minutes (De la nada al nirvana en cuestión de minutos) que se celebró en Google I/O 2013. Consulta también la documentación para desarrolladores sobre Cloud Endpoints. Además, las instancias de Google Cloud Endpoints se muestran en varios ejemplos de soluciones, incluidas Aplicación web, Mobile Shopping Assistant y Mobile Backend Starter.

API de Compute Engine

La de API de Compute Engine proporciona un método de programación para controlar un conjunto de máquinas virtuales en el entorno en la nube de Google. Se pueden iniciar y crear máquinas virtuales simples según las especificaciones necesarias. Asimismo, se pueden crear de antemano imágenes de las máquinas para reducir el tiempo necesario para iniciarlas. El escalado automático se puede lograr a través de comprobaciones periódicas de la carga de la máquina. Si utilizas colas de tareas de App Engine, puedes dividir el trabajo y escalar tu entorno de procesamiento en horizontal. Las ventajas de Google Compute Engine se pueden aprovechar para ofrecer procesamiento por lotes o personalizado para casi cualquier carga de trabajo.

Para obtener más información sobre cómo coordinarte con la API de Compute Engine, consulta nuestro ejemplo de servidor para juegos con carga equilibrada. Ten en cuenta que, aunque en el ejemplo se utilizan los juegos como caso, muchos patrones de diseño se pueden generalizar a otros casos.

Almacenamiento de datos compartidos

El almacenamiento de datos está profundamente integrado en Google Cloud Platform. Hay un servicio para satisfacer las necesidades de tu aplicación, desde el almacenamiento de archivos básico al almacenamiento de objetos de transacción:

  • Cloud Storage ofrece un servicio de almacenamiento para objetos y para archivos con un tamaño que llega a los terabytes.
  • Cloud SQL proporciona una base de datos SQL relacional basada en MySQL.
  • Cloud Datastore (el almacén de datos de App Engine) proporciona un almacén de datos de objetos NoSQL sin esquema, con un motor de consultas y con transacciones atómicas.

Todos y cada uno de estos servicios comparten un tema común, la accesibilidad. Están creados para que se pueda acceder a ellos desde cualquier lugar de Cloud Platform y también desde fuera. Todos los servicios representan una variedad de opciones de almacenamiento para los desarrolladores de aplicaciones. Tanto si necesitas la simplicidad de una implementación SQL pura como si optas por la escalabilidad de NoSQL, hay una opción de base de datos disponible para tu aplicación.
Google Cloud Platform gestiona automáticamente la autenticación y la autorización. De manera predeterminada, las aplicaciones tienen autorización para acceder a otros recursos que pertenezcan a un mismo proyecto. La cuenta de servicio es un tipo de cuenta con función que es específica para el proyecto y que permite el acceso de la aplicación a los recursos. Fuera de Cloud Platform, las cuentas de servicio basadas en certificados están disponibles para permitir un acceso seguro desde ubicaciones remotas. Las cuentas de servicio se pueden crear y revocar fácilmente desde dentro del proyecto en la nube.

Módulos de App Engine

Los módulos de App Engine permiten a los equipos dividir las aplicaciones en componentes con diferentes requisitos para:

  • el escalado,
  • las implementaciones,
  • el control de versiones,
  • la configuración del rendimiento.

Las aplicaciones estándar a gran escala se dividen en servicios lógicos que posibilitan los diversos aspectos del negocio. Estos componentes pueden ser compatibles con una serie de áreas, como, por ejemplo, un sitio corporativo, las aplicaciones web, el acceso para móviles, el acceso a la API de cliente y los portales de administración. Normalmente, estos diferentes sitios se encargan de diferentes requisitos de niveles de servicio, de tráfico previsto y de procesamiento. Cumplir estos requisitos en un entorno administrado estándar suele obligar al desarrollador a elegir el nivel de rendimiento más alto para todas las aplicaciones. Los módulos de App Engine dan vía libre a la posibilidad de personalizar una aplicación en unidades lógicas sin sacrificar las ventajas de un entorno totalmente administrado, incluido el escalado automático opcional, el control de versiones de implementación automático y la división del tráfico.

El resultado final es un entorno creado para varios equipos o para varios lenguajes de desarrollo. Una aplicación de App Engine puede satisfacer las diversas necesidades de una aplicación de gran tamaño.

Resumen

Google Cloud Platform ofrece una serie de herramientas y de servicios para administrar aplicaciones a gran escala. Al combinar App Engine y Compute Engine, se obtiene una completa plataforma de procesamiento que admite un conjunto variado de cargas de trabajo. El conjunto de servicios, que incluye Cloud Storage, Cloud Datastore, Cloud SQL y las colas de tareas, proporciona los elementos básicos para varias necesidades de desarrollo. Además, las API y las herramientas de administración de Cloud Platform contribuyen a aliviar el coste de administración general hasta de las aplicaciones más grandes.
