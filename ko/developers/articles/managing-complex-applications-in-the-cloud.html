Cloud에서 복잡한 애플리케이션 관리하기

Google Cloud Platform 애플리케이션 개발의 실제

오늘날 점점 더 많은 비즈니스 애플리케이션에서 특정 수준의 복잡성을 요구하고 있지만 단일 호스팅 서비스로 이를 해결하는 것은 쉬운 일이 아닙니다. 하나의 애플리케이션에는 여러 논리적 서비스, 다양한 프로그래밍 언어, 데이터 저장소 유형에 따른 상이한 요구사항, 광범위한 런타임 요구사항이 존재합니다. 또한 팀은 대부분 광범위하고 다양한 영역의 전문성과 경험을 갖춘 개발자로 구성됩니다. 그러나 요구사항이 너무나도 빠르게 변화하기 때문에 가장 규모가 큰 팀에서도 감당하기 어려운 수준의 복잡함이 야기되는 경우가 종종 발생합니다.

소개

Google Cloud Platform은 가장 복잡한 애플리케이션 요구사항까지 관리할 수 있는 완벽한 도구 모음을 제공하기 위해 계속해서 향상되어 왔습니다. 완전히 관리되는 애플리케이션에서 맞춤형 가상 머신에 이르기까지 Cloud Platform은 개발자에게 관리 간접 비용과 기술적 요구 간의 균형을 맞출 수 있는 유연성을 제공합니다. Google Cloud Platform에서는 애플리케이션을 광범위하고 다양한 기술적 비즈니스 요구사항이 적용된 별도의 구성요소로 쉽게 개발할 수 있습니다. 이러한 구성요소는 자체적인 성능 구성, 예산, 승인 및 버전 차별화 기능을 포함하는 동시에 공통의 리소스 모음을 공유할 수 있습니다. 공유 리소스를 사용하여 개별 애플리케이션은 기존 API를 통해 통신할 때보다 더욱 긴밀하게 통합될 수 있습니다. 또한 구성요소는 통합된 보안 모델과 Cloud 프로젝트[1]의 중앙 집중식 관리의 이점을 활용할 수 있습니다. Cloud Platform을 이용하여 개발하는 경우 점점 더 심해지는 애플리케이션의 복잡함을 관리할 수 있는 다양한 도구를 제공받을 수 있습니다.

처음부터 새로운 애플리케이션을 제작하든 아니면 기존의 맞춤 애플리케이션[2]을 클라우드로 이전하든 Google Cloud Platform은 비즈니스에 적합한 수준의 성능과 기능을 제공해 드릴 수 있습니다.

이 솔루션에서는 다음을 위한 최적의 도구를 제시해 드립니다.

  • 개발자의 다양한 전문성
  • 데이터 저장소와 관련된 다양한 요구사항
  • 양방향 및 일괄 처리 요구사항
  • 맞춤 분석 엔진
  • 논리적으로 독립된 비즈니스 구성요소

범위 및 시나리오

이 문서는 Google Cloud Platform에서의 대규모 애플리케이션 배포 및 확장을 관리하는 데 관심이 있는 솔루션 설계자 및 CTO를 위해 작성되었습니다. 이 문서에 제시된 예는 대규모 애플리케이션 배포 시 발생하는 여러 복잡성 문제를 해결하는 방법을 보여줍니다. 또한 여기에서는 플랫폼 도구를 소개하고 자세한 관련 정보가 있는 페이지로 연결되는 링크를 제공합니다. 이 문서에서 제시되는 샘플은 앱과 관련된 고유한 기능을 상세하게 설명하기보다는 일반적인 복잡성을 해결하는 데 중점을 두기 위해 의도적으로 단순하게 구성했습니다.

Google Cloud Platform 사용 시 클라우드에서 복잡한 애플리케이션을 어떻게 관리할 수 있는지 설명하기 위해 동영상 공유 커뮤니티를 구축하는 과정을 다룬 실제 시나리오를 살펴볼 것입니다. 이 시나리오에서 사용자는 휴대전화를 사용하여 동영상을 업로드합니다. 콘텐츠는 여러 형식으로 재생할 수 있도록 트랜스코딩되며 최종적으로 다른 사용자들이 시청하고 의견을 제공할 수 있도록 웹 커뮤니티 사이트에 게시됩니다. 시나리오의 데모 애플리케이션에는 다음과 같은 구성요소가 포함됩니다.

  • 동영상 업로드를 위한 Android 애플리케이션 및 모바일 백엔드
  • 동영상 트랜스코딩 파이프라인
  • 동영상 시청 및 의견 제공을 위한 커뮤니티 웹사이트
  • Hadoop을 통한 의견 분석

논의되는 기술적 과제는 다음과 같습니다.

  • 여러 백엔드 언어
  • 여러 독립 개발자 또는 팀
  • 관리형 애플리케이션과 맞춤 바이너리의 조합
  • 데이터 저장소와 관련된 여러 요구사항
  • 일괄 처리를 위한 동적 확장

이 외에 2013년 Google I/O에서 제공했던 라이브 데모도 시청할 수도 있습니다.

  • Apache Hadoop
  • FFmpeg

웹 서비스 배포 및 확장의 관리

일반적인 웹 자산을 배포하고 관리하는 경우 기본적인 네트워크, 시스템 및 서비스를 관리하는 데 상당한 간접 비용이 발생합니다. App Engine은 완벽하게 관리되는 애플리케이션 플랫폼을 제공하여 이러한 우려를 없애 주며, 수요에 따라 자동으로 리소스를 확장하거나 축소합니다. 버전 차별화는 개발자가 롤백을 손쉽게 처리하고 출시 버전 간에 고객 트래픽을 나누며 새로운 출시 과정을 천천히 진행할 수 있게 해 주는 또 하나의 강력한 기능입니다. 웹 작업 부하와 관련해서도 App Engine은 개발자가 최고의 애플리케이션을 제작하는 데 완전히 집중할 수 있도록 도와줍니다.

App Engine에서 동영상 공유 사이트의 웹 영역은 위와 같은 고유한 이점을 활용할 수 있도록 구축됩니다. 이 구성요소는 가장 널리 사용되는 웹 언어인 PHP로 작성되었으며, App Engine의 PHP 지원 기능은 자동 확장, 버전 차별화 및 성능에 웹 전용 언어를 통한 신속한 개발을 결합함으로써 얻게 되는 모든 이점을 제공합니다. 대부분의 기존 PHP 애플리케이션은 최소한의 변경만으로도 App Engine 환경에서 실행할 수 있습니다.

대부분의 웹 개발 팀이 MySQL에 익숙하므로 Google은 이와 매우 친숙한 데이터베이스 기술인 Google Cloud SQL을 사용하기로 결정했습니다. Cloud SQL은 MySQL 기능 및 SQL 프로토콜과 호환되므로 기존 작업 부하를 간단하게 변환할 수 있습니다. 또한 Google은 눈에 보이지 않는 고가용성 및 백업의 복잡성에 대해 논의합니다. 샘플에서 Cloud SQL은 사용자 데이터 및 동영상 메타데이터 저장을 위한 기본 저장소로 사용되었습니다.

동영상 파일과 미리보기 이미지의 저장소로 Google Cloud Storage를 선택했습니다. Cloud Storage는 대용량 파일을 지원하며 애플리케이션 클라이언트에 파일을 직접 전달할 수 있습니다. 또한 동영상 콘텐츠를 전역적으로 배포할 수 있도록 내장형 에지 캐시 옵션을 제공합니다. 업로드 시에는 시간 제한이 적용된 쓰기 가능한 URL을 제공함으로써 클라이언트에서 바로 파일을 업로드할 수 있게 해 줍니다. 따라서 파일을 저장소에 전달하기 위해 소중한 애플리케이션 리소스를 소비할 필요가 없습니다.

사이트의 사용자 활동 즉, 의견 올리기와 관련하여 Google에서는 이러한 활동이 상당히 늘어날 것으로 예상했으며 App Engine의 NoSQL 데이터베이스인 고성능 복제 데이터 저장소를 사용하기로 선택했습니다. HRD는 순수한 SQL 솔루션과는 비교할 수 없는 수준의 확장성을 제공하며 대용량 데이터에 액세스하는 데 적합한 제품입니다. 이러한 선택을 통해 새로 출시된 Cloud Datastore를 사용할 기회를 얻고 Cloud Platform 전반 및 외부에서 발생하는 사용자 활동에 액세스할 수 있게 되었습니다.

일괄 처리 파이프라인의 관리

대부분의 대규모 웹사이트에서는 특정 수준의 일괄 처리 기능을 요구합니다. 작업 부하에는 업로드된 콘텐츠 처리, 사용자 데이터 업데이트, 외부 알림 관리 또는 데이터 분석이 포함될 수 있습니다. 비즈니스 요구와 상관없이 일괄 처리 파이프라인은 구축, 확장 및 유지 관리가 본질적으로 쉽지 않습니다. Google Cloud Platform은 확장 가능하고 안정적인 처리 파이프라인을 구축하는 작업을 간소화할 수 있도록 지원하는 도구와 서비스를 제공합니다. 프로그래밍 가능한 컴퓨팅 환경에서 광범위한 저장소 옵션에 이르기까지 이 플랫폼은 분산형 일괄 처리에 맞게 조정되었습니다.

이 시나리오에서는 일괄 처리 파이프라인을 사용하여 애플리케이션에 업로드된 동영상 콘텐츠를 관리합니다. 대부분의 동영상 공유 사이트는 업로드된 콘텐츠에 일괄 처리를 사용하여 트랜스코딩, 워터마킹 및 디지털 권한 관리를 비롯한 모든 작업을 수행합니다. 이러한 서비스에 대한 수요는 단발적인 콘텐츠 업로드에 비해 다소 안정적인 소비 행동에 따라 극심하게 변동하는 경향이 있습니다. 또한 동영상 파이프라인의 다양한 구성요소는 엄격한 PaaS(서비스형 플랫폼) 환경에 적합하지 않은 맞춤 애플리케이션이나 코드를 요구하는 경우가 많습니다. 이 두 요인 모두가 Compute Engine이 왜 이상적인 선택인지를 보여줍니다. 일반적인 데이터 센터 또는 관리형 호스팅을 확장하는 경우 미리 용량을 구입하거나 시간 단위 블록에 대한 비용을 지불해야 합니다. Google Compute Engine은 인스턴스를 단 몇 초 만에 빠르게 가동하고 종료할 수 있도록 해 주므로 이러한 유형의 작업 부하에 매우 이상적인 제품입니다. 이뿐 아니라 Compute Engine은 확장이 진행되는 동안 매우 일관된 성능을 제공합니다.

Google 환경에서는 App Engine과 Compute Engine을 함께 사용하여 수요에 따라 확장형 동영상 트랜스코딩 서비스를 가동합니다. 또한 App Engine에서 Python으로 작성된 통합 엔진을 활용하여 사용 가능한 JSON API 및 작업 대기열을 통해 조정된 트랜스코딩 액티비티를 사용하여 Compute Engine의 인스턴스 동작을 제어합니다. 이 설정을 사용하면 수요 및 기존 부하에 맞게 용량을 동적으로 할당할 수 있습니다. 또한 정기적으로 실행되는 스크립트에서는 환경의 부하를 확인하고 필요 없는 용량을 줄일 수 있습니다. 이는 최근 변경된 Compute Engine의 시간 단위 결제 방식을 고려할 때 강력한 경제적 이점을 제공합니다. 작업 대기열은 동영상 트랜스코딩 인스턴스 전반에서 요청을 수락하고 임시 저장소에 저장하며 분배할 수 있도록 해 줍니다.

신속한 모바일 배포 관리

Google Cloud Platform은 신속한 모바일 애플리케이션 개발을 위한 도구 모음을 제공합니다. Cloud Endpoints와 App Engine의 조합은 백엔드 서버 기술을 외부 소비용 API로 사용할 기회를 제공하는 플랫폼으로 기능합니다. Cloud Endpoints 없이 모바일 애플리케이션을 기존 웹사이트에 추가하기 위해서는 유연한 API와 강력한 인증에 대비한 어느 정도의 계획이 필요합니다. Endpoints는 인증 및 API 생성을 자동으로 처리해 주므로 개발자가 애플리케이션 기능을 구현하는 데 더 많은 시간을 할애할 수 있습니다. 또한 App Engine을 사용하면 API 서버의 확장과 지속적인 유지 관리 작업을 간소화할 수 있습니다.

API 서버는 모바일 애플리케이션을 동영상 공유 사이트에 추가할 수 있도록 App Engine에 구축되었으며 인증 및 신속한 API 개발을 지원하기 위해 Google Cloud Endpoints를 사용했습니다. 또한 여러 비즈니스 구성요소 간의 유연한 언어 선택을 보여줄 수 있도록 자바에 백엔드를 구현하기로 결정했습니다. 모바일 백엔드에서는 모바일 애플리케이션에서 Cloud Storage로 동영상 콘텐츠를 업로드하고 Cloud SQL에 메타데이터를 생성하며 작업 대기열을 통해 트랜스코딩 서비스에 통합하는 작업을 지원합니다.

모바일 동영상 애플리케이션은 생성된 Cloud Endpoints 라이브러리를 활용하는 Android에서 작성되었으므로 인증 및 동영상 업로드 기능을 신속하게 통합할 수 있습니다. 또한 App Engine API 서버에서 추가적인 지연 시간이나 처리 작업이 발생하지 않게 하기 위해 동영상이 Cloud Storage에 바로 업로드됩니다. 인증은 Cloud Endpoints를 통해 원활하게 처리되며 동영상 콘텐츠를 저장할 수 있도록 Cloud Storage 버킷에 권한이 위임됩니다. 권한 위임은 버킷에서 제공되는 옵션으로 애플리케이션 사용자가 파일을 직접 업로드할 수 있게 해 줍니다. 간소화된 인증 및 API 생성 기능을 함께 사용함으로써 완벽한 모바일 미디어 솔루션을 신속하게 개발할 수 있었습니다.

사용자 데이터 분석 관리

애플리케이션의 사용자 활동을 분석하면 다양한 시스템과 소프트웨어에서 사용되는 엄청난 양의 데이터를 확보하여 의미 있는 결과를 얻을 수 있습니다. 대부분의 분석이 단발적인 특성이 있으므로 제공된 환경은 여러 특정 시간 동안 유휴 상태로 머무를 수 있습니다. 그러나 분석 작업 수행 시 이러한 시스템에서는 적절한 시간 프레임 내에 분석을 완료하기 위해 방대한 컴퓨팅 리소스와 저장소를 소비할 수 있습니다. 이러한 특성은 Compute Engine이 왜 이상적인 제품인지를 보여줍니다. 늘어나는 작업 부하를 처리하기 위한 신속한 인스턴스 가동 능력과 유휴 시의 축소 기능을 사용하면 크고 작은 분석 작업 모두를 수요에 기반하여 실행할 수 있습니다. 또한 확장을 위한 용량을 확보함으로써 일반적인 데이터 처리 환경에 수반되는 병목현상을 방지할 수 있습니다.

동영상 애플리케이션의 예상 분량의 의견을 처리하기 위해 Compute Engine에서 실행 중인 Hadoop 설치를 사용하기로 결정했습니다. 다시 강조하지만 Compute Engine API를 사용하면 다양한 크기의 Hadoop 클러스터를 실행하여 데이터를 손쉽게 처리할 수 있습니다. 이 사례에서는 애플리케이션에 연결된 Cloud Datastore의 의견 데이터를 가져와 동영상에 게시된 의견과 관련하여 간단한 감성 계산을 수행합니다. Cloud Datastore 데이터는 같은 프로젝트에 속한 App Engine 및 Compute Engine 인스턴스 간에 자동으로 공유됩니다. 부정적 또는 긍정적인 전체 감성에 대한 컴퓨팅의 최종 결과는 동영상에 연결된 Datastore 항목에 다시 기록됩니다.

Google은 데이터 분석을 위한 몇 가지 대안을 제공합니다. 특히 감성 분석과 관련하여 Google Prediction API는 머신 학습 환경에 REST 인터페이스를 제공하며, 이 인터페이스는 여러 옵션 중에서도 특히 사용자 데이터에 대한 감성 분석을 수행합니다. 좀 더 일반적인 데이터 분석의 경우 Google Big Query에서 맞춤 개발로 인한 간접 비용이 없는 테라바이트 단위의 데이터를 처리할 수 있는 엔진을 제공합니다. 맞춤형 Datastore 처리를 위한 또 다른 이상적인 대안은 App Engine Map Reduce입니다.

복잡성 관리

애플리케이션의 여러 논리적 서비스를 고려하여 Google Cloud Platform의 몇 가지 핵심 기능을 최대한 사용했습니다. 이 플랫폼은 이러한 모든 기능을 함께 사용하여 클라우드의 복잡한 비즈니스 애플리케이션을 지원하기 위한 강력한 기반을 형성합니다.

Cloud Endpoints

Cloud Endpoints는 App Engine 애플리케이션에서 REST API 및 클라이언트 라이브러리를 생성하여 웹 클라이언트와 모바일 클라이언트 액세스를 모두 간소화하는 데 사용되는 도구 및 서비스 모음입니다. 이 조합을 사용하면 Android 애플리케이션에서 신속하게 API 서버를 시작할 수 있습니다. Cloud Endpoints는 App Engine 애플리케이션과의 통신을 간소화함으로써 Cloud Datastore, Cloud Storage, Mail, 작업 대기열과 같은 현재 사용 가능한 서비스의 이점을 활용할 수 있게 해 줍니다. 이 제품은 보이지 않는 곳에서 REST API 호출을 백엔드 애플리케이션의 메소드 호출로 변환하고 OAuth 2.0 인증을 통해 각 호출에 보안을 설정합니다. 현재 Eclipse, Android Studio 및 Apache Maven을 포함한 기존 개발 환경에서는 프로젝트 내에 클라이언트 라이브러리를 생성할 수 있는 기능을 제공합니다. Cloud Endpoints 기술을 사용하면 API 개발로 인한 간접 비용을 없앨 수 있으며 개발자는 자신에게 가장 중요한 일인 '제품'에 집중할 수 있습니다.

Google Cloud Endpoints를 시작하기 위해 2013년 Google I/O에서 제공되었던 From Nothing to Nirvana in Minutes(몇 분 만에 무에서 열반의 경지로) 세션과 Cloud Endpoints 개발자 문서를 참조할 수 있습니다. 또한 Google Cloud Endpoints는 웹 앱, 모바일 쇼핑 도우미 및 Mobile Backend Starter를 포함한 여러 솔루션 샘플에 포함되어 있습니다.

Compute Engine API

Compute Engine API는 Google Cloud 환경에서 가상 머신 그룹을 제어할 수 있도록 프로그래밍 방식의 메소드를 제공합니다. 따라서 기본적인 가상 머신을 실행한 다음 필요한 사양에 맞게 시스템을 보완할 수 있습니다. 또한 머신 이미지를 미리 생성하여 실행에 필요한 시간을 단축할 수 있습니다. 자동 확장은 시스템 부하를 주기적으로 점검함으로써 구현할 수 있습니다. App Engine 작업 대기열을 사용하면 작업을 나누고 이에 맞게 처리 환경을 수평으로 확장할 수 있습니다. Google Compute Engine의 이점을 활용하면 거의 모든 작업 부하를 일괄적으로 혹은 맞춤 형식으로 처리할 수 있습니다.

Compute Engine API 조정에 대한 자세한 내용은 부하 분산 게임 서버 샘플을 참조하세요. 샘플에서는 게임을 시나리오로 사용하지만 대부분의 설계 패턴은 다른 시나리오에서도 일반적으로 사용할 수 있습니다.

공유 데이터 저장소

데이터 저장소는 Google Cloud Platform에 긴밀하게 통합되며, 기본 파일 저장소에서 트랜잭션 객체 저장소에 이르기까지 애플리케이션의 요구에 부합하는 다음과 같은 서비스가 제공됩니다.

  • Cloud Storage는 최대 수 테라바이트 크기의 객체 및 파일을 위한 저장소 서비스를 제공합니다.
  • Cloud SQL은 MySQL 기반의 관계형 SQL 데이터베이스를 제공합니다.
  • Cloud Datastore(App Engine Datastore)은 NoSQL 스키마 제외 객체 데이터 저장소와 쿼리 엔진 및 원자적 트랜잭션을 제공합니다.

이러한 각 서비스는 액세스 가능성이라는 공통된 주제를 공유합니다. 또한 이 서비스는 Cloud Platform은 물론 외부 어디에서도 액세스할 수 있도록 제작되었습니다. 모든 서비스는 애플리케이션 개발자를 위한 다양한 저장소 옵션을 의미합니다. 순수한 SQL 구현의 간소함이 필요하든 NoSQL의 확장성을 선택하든, 애플리케이션에는 사용 가능한 데이터베이스 옵션이 있습니다.
인증 및 승인은 Google Cloud Platform에서 자동으로 처리합니다. 기본적으로 애플리케이션에는 동일한 프로젝트에 속하는 다른 리소스에 액세스할 수 있는 권한이 있습니다. 서비스 계정은 프로젝트에 고유한 역할 계정 유형으로 애플리케이션에서 리소스에 액세스할 수 있게 해 줍니다. Cloud Platform 외부에서는 인증 기반 서비스 계정을 사용하여 원격 보안 액세스를 실행할 수 있습니다. 서비스 계정은 클라우드 프로젝트 내에서 쉽게 생성하고 취소할 수 있습니다.

App Engine 모듈

App Engine 모듈을 통해 팀에서는 다음에 대한 별도의 요구사항이 지정된 구성요소로 애플리케이션을 나눌 수 있습니다.

  • 확장
  • 배포
  • 버전 차별화
  • 성능 설정

일반적인 대규모 애플리케이션은 비즈니스의 다양한 측면을 지원하는 논리적 서비스로 나뉩니다. 이러한 구성요소는 기업 사이트, 웹 애플리케이션, 모바일 액세스, 고객 API 액세스 및 관리 포털을 비롯한 여러 영역을 지원할 수 있습니다. 대개 이러한 다양한 사이트에는 서로 다른 서비스 수준, 예상 트래픽 및 처리 요구사항이 있습니다. 일반적인 관리 환경에서 이러한 요구사항을 충족하기 위해서는 보통 개발자가 모든 애플리케이션에 대해 가장 높은 수준의 성능을 선택할 수밖에 없습니다. App Engine 모듈은 애플리케이션을 논리적 단위로 맞춤설정할 수 있는 기능을 사용할 기회를 제공하는 한편 선택적인 자동 확장, 자동 배포 버전 차별화 및 트래픽 분할과 같은 완전히 관리되는 환경의 이점 또한 제공합니다.

따라서 여러 팀과 개발 언어를 위한 환경이 구축됩니다. 한 개의 App Engine 애플리케이션으로 대규모 애플리케이션의 다양한 요구사항을 충족할 수 있습니다.

요약

Google Cloud Platform은 대규모 애플리케이션 관리를 위한 여러 도구 및 서비스를 제공합니다. App Engine과 Compute Engine의 조합은 다양한 작업 부하 모음을 지원하는 완벽한 컴퓨팅 플랫폼으로 기능합니다. Cloud Storage, Cloud Datastore, Cloud SQL, Cloud Storage 및 작업 대기열을 포함하는 서비스 모음은 다양한 개발 요구를 충족하는 제작 도구를 제공합니다. 또한 Cloud Platform API 및 관리 도구는 애플리케이션의 크기와 상관없이 간접 관리 비용을 줄여 줍니다.
