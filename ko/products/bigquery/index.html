BigQuery

BigQuery로 클라우드에서 빅 데이터를 분석해 보세요. 테라바이트급 데이터 세트에서 SQL과 비슷한 쿼리를 단 몇 초 만에 빠르게 실행할 수 있습니다. 확장 가능하고 사용이 용이한 BigQuery는 데이터에 대한 실시간 통계를 제공합니다.

지금 사용해 보기

기능

모두 백그라운드에서 실행

쿼리는 백그라운드에서 비동기식으로 실행되며 폴링하여 상태를 확인할 수 있습니다. Google Cloud Console에서 나머지 Cloud Platform 리소스와 함께 쿼리 및 작업 기록에 액세스할 수 있습니다.

손쉬운 데이터 가져오기

Google Cloud Storage를 사용하여 데이터를 일괄 로드하거나 최대 1초당 1,000개 행 처리 속도로 한번에 스트리밍해 보세요. 실시간 대용량 로그 기록 또는 트랜잭션 기록을 처리하고, 데이터를 빠르고 쉽게 분석할 수 있습니다.

저렴한 빅 데이터

Google에서 '빅 데이터'라고 한다면 이는 정말 큰 단위의 데이터를 말합니다. 매달 처리되는 데이터 중 최초 100GB는 무료입니다. Google에서는 사용자를 대신하여 수천 테라바이트를 저장하고 쿼리합니다. Google의 가격 책정 구조는 단순하고 투명하며, 고객은 가격 구조를 전체적으로 파악하고 원하는 대로 선택할 수 있습니다. 필요로 하는 저장소와 실행하는 쿼리만큼만 비용을 지불하면 됩니다.

적절한 인터페이스

관리자용과 개발자용 인터페이스를 분리하여 필요한 도구에 액세스할 수 있도록 했습니다. Google API 콘솔을 사용하여 프로젝트와 데이터 세트 수준 모두에서 액세스를 제어할 수 있습니다. BigQuery를 사용하면 팀 규모에 상관없이 원하는 방식으로 작업을 할 수 있습니다.

가격 책정

BigQuery는 열 형식의 데이터 구조를 사용합니다. 즉, 특정 쿼리에서 전체 표가 아닌 각 열에서 처리된 데이터에 대해서만 비용이 청구됩니다. 매달 처리된 데이터 중 처음 100GB는 무료입니다.

가격은 2014년 4월 1일부터 적용됩니다.

문서 및 리소스

광범위한 문서 자료와 가이드를 통해 BigQuery를 애플리케이션에 통합하는 방법을 익힐 수 있습니다. 데이터 일괄 로드, 삽입 항목 스트리밍 및 비동기식 쿼리 실행 방법에 대해 알아보세요. 다른 Cloud Platform 서비스에서도 빠르게 BigQuery의 기능에 액세스할 수 있습니다.

리소스

  • BigQuery란 무엇인가요?

    대규모 데이터 세트를 쿼리할 때 올바른 하드웨어와 인프라를 사용하지 않으면 시간이 많이 소모되고 가격이 비싸질 수 있습니다. Google 인프라의 처리 능력을 기반으로 한 Google BigQuery는 추가 전용 표에서 SQL과 유사한 매우 빠른 쿼리를 사용하여 이러한 문제를 해결했습니다. BigQuery로 데이터를 이동하기만 하면 됩니다. 힘든 작업은 Google에서 처리합니다.

  • BigQuery에 데이터 로드

    데이터를 쿼리하려면 먼저 BigQuery에 로드해야 합니다. 작업을 사용하여 데이터를 일괄 로드하거나 기록을 개별적으로 스트리밍할 수 있습니다.

  • 데이터 쿼리

    쿼리는 BigQuery의 SQL 언어로 작성됩니다. BigQuery는 동기식 및 비동기식 쿼리 메소드를 모두 지원합니다. 두 메소드 모두 작업으로 처리되지만 '동기식' 메소드는 반환하기 전에 작업이 완료될 때까지 대기하는 타임아웃 값을 노출합니다.
