Prediction API

Google의 기계 학습 알고리즘을 사용하여 익숙한 RESTful 인터페이스에서 데이터를 분석하고 향후 결과를 예측해 보세요.

지금 사용해 보기

기능

보유 데이터 활용하기

과거 데이터를 사용하여 향후 트렌드를 예측해 보세요. Prediction API를 사용하여 메시지 전달 방식을 지정하고 스팸을 감지하고 사용자에게 제품을 추천할 수 있습니다. Google의 강력한 인프라가 고객을 대신하여 데이터를 분석합니다.

신속성과 신뢰성

데이터는 Cloud Storage를 통해 여러 데이터 센터에 복제됩니다. 대부분의 예측 쿼리는 200밀리초 이내에 수행되며, 성능 개선도 가능합니다. 성능에 대한 도움말은 이 문서를 참조하세요.

Cloud 통합

Google의 서비스들은 함께 작동하도록 설계되었습니다. Google BigQuery의 결과를 사용하거나 Cloud Storage에서 최대 2.5GB의 데이터 세트를 가져와 예측 모델을 만들어 보세요. Google App Engine에서 바로 Prediction API를 사용할 수도 있습니다.

강력한 개발 도구

Prediction API는 App Engine과 통합 가능하며, Python, JavaScript, .NET과 같은 여러 인기 언어의 라이브러리에서 RESTful API를 사용할 수 있습니다. 전체 목록은 이 문서를 참조하세요.

예제 및 지원

애플리케이션 예제를 확인하여 Prediction API에 대해 자세히 알아보세요. 더 많은 아이디어를 얻으려면 데모와 추천 애플리케이션을 둘러보세요.

유연한 가격 책정 방식

처음 6개월 동안은 제한된 사용량을 무료로 제공합니다. 그 이후에는 Prediction API의 합리적인 가격 책정 방식에 따라, 가용성 99.9%의 서비스 수준 계약으로 서비스가 제공됩니다. Cloud의 다른 제품과 마찬가지로 사용한 만큼만 지불하면 됩니다.

가격 책정

무료 할당량

처음 6개월 동안은 Google Cloud Console 프로젝트당 다음 한도까지 무료 사용이 가능합니다. 이 무료 할당량은 결제를 사용 설정한 경우에도 6개월 기간이 만료될 때까지 계속 적용됩니다.

  • 사용량 한도:
    • 예측: 일일 예측 100건
    • 호스팅된 모델 예측: 호스팅된 모델의 경우 모든 모델에 사용자당 일일 예측 100건의 사용량 한도가 적용됩니다.
    • 학습: 일일 학습된 데이터 5MB
    • 스트리밍 업데이트: 일일 스트리밍 업데이트 100건
  • 총 한도: 예측 20,000건
  • 만료: Google Cloud Console에서 프로젝트를 위해 Google 예측을 활성화한 후 6개월이 지나면 무료 할당량이 만료됩니다.

Google Cloud Storage를 사용하는 경우 일부 Prediction API 사용량이 달라질 수 있으며, Prediction API와는 다른 기준으로 가격이 책정됩니다.

유료 사용량

유료 사용량은 99.9%의 가용성 서비스 수준 계약으로 제공됩니다. SLA에 대한 자세한 내용은 여기를 참조하세요.

Google Cloud Console에서의 사용량을 모니터링해 보세요. 결제를 사용 설정하지 않은 프로젝트에서 무료 사용 할당량을 초과하면 API를 사용할 수 없게 됩니다. 무료 할당량을 초과하면 다음과 같은 가격 및 한도가 적용됩니다.

  • 기본 요금: Google Cloud Console 프로젝트당 $10의 월정액 요금
  • 사용 요금:
    • 예측:
      • 월 10,000건의 예측: $0.00
      • 월 10,001건 이상의 예측: 처음 10,000건을 초과하는 예측 1,000건당 $0.50
    • 학습:
      • 일괄 학습된 MB당 $0.002(각 데이터 세트의 최대 크기: 2.5GB)
      • 스트리밍 업데이트 0~10,000건: $0.00
      • 스트리밍 업데이트 10,001건 이상: 처음 10,000건을 초과하는 업데이트 1,000건당 $0.05
  • 사용량 한도:
    • 일일 40,000건 이상의 예측이 필요한 경우에는 문의해 주세요.
    • Google Prediction에서는 자체 모델의 프로젝트당 일일 예측 2,000,000건 이하의 기본 제한이 적용됩니다.
    • 호스팅된 모델의 경우 모든 데모 모델에서 사용자당 일일 예측 100건의 사용량 제한이 있습니다. 개발자는 자신이 만든 모델에 대해 유료 사용량 한도를 더 높이 설정할 수 있습니다. 자세한 내용은 각 모델의 문서를 참조해 주세요.

중요: 이러한 요금에는 학습 데이터 보관에 필요할 수 있는 Google Cloud Storage 요금이 포함되어 있지 않습니다. 발생할 수 있는 추가 요금을 확인하려면 Google Cloud Storage의 가격 책정 표를 참조해 주세요.

문서 및 리소스

Prediction API의 학습 방법과 이러한 예측 기능을 애플리케이션에 통합하는 방법에 대해 알아보세요. 도구 및 리소스에는 다른 Cloud Platform 제품과 함께 Prediction API를 사용하는 방법을 보여주는 샘플 코드와 안내가 포함되어 있습니다.

리소스

  • Google Prediction API란 무엇인가요?

    Google의 클라우드를 기반으로 한 기계 학습 도구는 데이터를 분석하여 고객 감정 분석, 스팸 감지, 메시지 전달 방식 결정, 상위 제품 판매 기회 분석, 문서 및 이메일 분류 등의 기능을 애플리케이션에 추가하는 데 도움이 될 수 있습니다.

  • 구매 예측

    맥주, 와인 ,치즈를 판매하는 사이트에서 구매 기록을 바탕으로 방문자가 와인에 관심이 있을지 여부를 예측하고자 한다고 가정해 보겠습니다. 이 경우 세 가지 특징으로 학습 데이터를 만들 수 있습니다. 즉, 고객이 디저트용 고급 치즈를 구입한 횟수, 고객이 와인을 구입한 적이 있는지 여부(있는 경우 1, 없는 경우 0), 고객이 맥주를 구입한 횟수 등입니다.

  • 감정 분석 모델 만들기

    이 문서는 Google Prediction API를 사용하여 기본 감정 분석 모델을 만드는 방법에 대해 설명합니다. 감정 분석 모델은 텍스트 문자열을 분석하고 입력한 라벨 중 하나로 이 문자열을 분류하는 데 사용합니다. 예를 들어 트윗을 분석하여 긍정적인지 부정적인지를 확인하거나, 이메일을 행복, 실망, 슬픔 등으로 분석하는 것입니다.
