Cloud Datastore

비관계형 데이터를 저장할 수 있는 스키마 없는 NoSQL의 관리형 데이터베이스를 사용해 보세요. Cloud Datastore는 필요에 따라 자동으로 크기를 조절하고 트랜잭션은 물론 SQL과 비슷한 강력한 쿼리를 지원합니다.

지금 사용해 보기

기능

SQL과 비슷한 쿼리, 스키마 없이 액세스

데이터 모델 및 마이그레이션에 대해 걱정할 필요가 없습니다. Cloud Datastore는 데이터의 기본 구조에 대해 고민할 필요가 없어 빠른 작업이 가능한 스키마 없는 저장소 서비스입니다. Cloud Datastore는 여러 항목에서 데이터를 검색하고 필요에 따라 이를 정렬할 수 있는 강력한 쿼리 엔진을 제공합니다.

관리형 데이터베이스

Cloud Datastore는 전체적으로 관리됩니다. 가용성 높고 일관성 있는 데이터베이스를 제공하기 위해 Google에서 자동으로 분할 및 복제를 처리합니다.

사용자에 따라 크기 자동 조절

Cloud Datastore는 필요에 따라 자동으로 크기를 조절합니다. 따라서 프로비저닝 및 로드 예상을 고민할 필요없이 애플리케이션 제작에 집중할 수 있습니다.

ACID 트랜잭션

Cloud Datastore는 최적의 동시 실행 제어를 사용하여 ACID 트랜잭션을 제공합니다. 애플리케이션에서 모두 성공이거나 모두 실패인 단일 트랜잭션으로 여러 데이터 저장소 작업을 실행할 수 있으므로 데이터 무결성이 보장됩니다.

내장형 중복 기능

단일 API 호출로 데이터가 자동으로 여러 데이터 센터에 복제됩니다. 핵심적인 부분에서부터 높은 가용성과 내구성이 기본으로 제공됩니다.

로컬 개발 도구

Cloud Datastore 개발 키트를 사용하면 전체 배포를 수행하지 않고도 로컬에서 애플리케이션을 개발, 테스트, 개선해 나갈 수 있습니다.

어디서나 데이터에 액세스

App Engine과 Compute Engine 모두를 활용해 솔루션을 제작하고 Cloud Datastore를 통합 지점으로 활용해 보세요. Cloud Datastore에 의해 노출되는 RESTful 인터페이스를 사용하여 배포 대상 모두가 간편하게 데이터에 액세스할 수 있습니다.

가격 책정

Google Cloud Datastore의 요금과 무료 할당량은 아래에 자세히 설명된 App Engine의 데이터 저장소 요금과 동일합니다. Cloud Datastore에는 Google App Engine에서 실행되는 게재 요소가 있으므로 인스턴스 시간 비용이 발생한다는 점에 유의해야 합니다.

리소스

  • Google Cloud Datastore 개요

    Google Cloud Datastore는 Google Cloud Compute Engine 인스턴스를 실행 중인 클라이언트나 다른 위치에서 Google Cloud Datastore에 액세스하고 관리하기 위한 도구 및 클라이언트 라이브러리 집합을 제공합니다. Google 지도, YouTube 및 Google Apps API를 호스팅하는 것과 동일한 Google 서비스에서 호스팅되는 RPC 기반 API를 제공하며, 가상 머신 또는 사내 자바 애플리케이션 서버에서 실행 중인 node.js 웹 앱처럼 대부분의 애플리케이션과 언어에서 사용할 수 있습니다.

  • Google Cloud Datastore 활성화

    다음에서는 Google Cloud Datastore API와 이 API에 연결된 할당량을 활성화하는 방법을 알아보고, 결제를 사용 설정하는 방법에 대해 설명합니다. 활성화 단계는 기존 설정 및 환경에 따라 달라질 수 있습니다.

  • Datastore의 주요 개념

    Datastore는 애플리케이션에 강력하고 확장 가능한 저장소를 제공하는 스키마가 없는 개체 데이터 저장소입니다. Datastore에는 예약된 시스템 중지 시간이 없으며, 매우 작은 단위의 트랜잭션, 높은 읽기/쓰기 가용성, 읽기와 상위 쿼리의 강력한 일관성 및 기타 모든 쿼리의 결과적 일관성 등과 같은 기능을 제공합니다.
