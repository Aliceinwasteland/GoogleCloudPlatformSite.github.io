Cloud Storage

뛰어난 내구성과 가용성을 자랑하는 개체 저장소 서비스를 사용해 보세요. 글로벌 에지 캐싱을 사용하여 사용자 위치에 상관없이 앱의 데이터에 빠르게 액세스할 수 있습니다.

지금 사용해 보기

기능

보안 및 안전

데이터는 여러 물리적 위치에 있는 중복 저장소를 통해 보호됩니다. OAuth와 세부적으로 지정된 액세스 컨트롤을 통해 강력하고 구성 가능한 보안을 형성합니다.

경쟁력 있고 유연한 가격 책정

데이터의 특성에 맞는 저장소 솔루션을 선택해 보세요. 표준 저장소와 DRA(가용성을 낮춘 내구성 있는 저장소) 중에 선택할 수 있습니다. 다른 Cloud Platform 서비스와 마찬가지로 사용한 만큼만 비용을 지불하면 됩니다.

모든 기능을 지원하는 API가 있는 개체 저장소

Cloud Storage는 간단한 API로 액세스가 가능하면서도, 고급 기능 모음으로 유연성과 성능은 강화되었습니다. 개체 버전 관리, 동시 업로드 및 CRC 기반의 무결성 확인을 통해 강력하고 정교한 애플리케이션 개발이 가능합니다. XML 또는 JSON을 직접 사용하여 액세스하거나 여러 인기 프로그래밍 언어의 라이브러리를 사용할 수 있습니다.

유연한 액세스

Cloud Storage에서 직접 정적 개체를 제공하거나, Cloud Storage를 다른 Google 제품의 데이터 저장소로 사용할 수 있습니다. BigQuery, App Engine 또는 다른 Cloud Platform 서비스에서 저장소에 액세스해 보세요. 속도, 유연성 및 OAuth 기반의 보안을 제공하는 인증된 RESTful API를 통해 데이터에 액세스할 수 있습니다.

리소스

  • Google Cloud Storage 개요

    Google Cloud Storage를 사용하면 데이터 양에 상관없이 언제 어디서나 데이터를 저장하고 가져올 수 있습니다. Google Cloud Storage는 개발자가 Google의 안정적인 고속 네트워크 인프라를 활용하여 안전하고 비용 효율적인 방식으로 데이터 작업을 수행할 수 있도록 간편한 프로그래밍 인터페이스를 제공합니다. 확장이 필요해지면 개발자는 Google 인프라 특유의 뛰어난 확장성을 누릴 수 있습니다.

  • Amazon S3에서 Google Cloud Storage로 마이그레이션

    이 페이지에서는 API를 사용하여 요청을 보내는 사용자를 위해 Amazon S3(Amazon Simple Storage Service)에서 Google Cloud Storage로 마이그레이션하는 방법에 대해 설명합니다.

  • 개념 및 기술

    이 섹션에서는 Google Cloud Storage를 효과적으로 사용하기 위해 알아야 하는 개념과 기술에 대해 설명합니다. 여기에서 설명하는 정보는 개발자를 위한 것이며, 개발자가 웹 프로그래밍에 익숙하고 HTTP 요청을 통해 웹 서비스를 사용하는 애플리케이션을 만드는 데 익숙하다는 가정 하에 쓰여진 내용입니다.
